01

プロンプトとは何か

AIへの「指示文」が結果の9割を決める

1-a. プロンプト = AIへの指示文

AIに何かをお願いするとき、あなたが入力する文章のことを「プロンプト」と呼びます。日本語で言えば「指示文」「お題」です。

AIは魔法のように「察してくれる」わけではありません。あなたが書いた指示の質が、そのまま回答の質になります。

例えるなら プロンプトは「レストランでの注文」と同じ。
「なんか美味しいもの」と頼むより、「辛さ控えめのグリーンカレー、ライス少なめ」と伝えた方が、確実に望みどおりの料理が出てくる。AIも全く同じです。

なぜ書き方で結果が劇的に変わるのか

1-b. Bad vs Good プロンプト 実例比較

まずは「同じ目的」なのに全く違う結果になる実例を見てみましょう。

例1: メールを書く

NG PROMPT
メールを書いて

→ 誰に？何の用件？どんなトーン？ AIは全部推測するしかない。的外れなメールが生成される。

VS
GOOD PROMPT
取引先の田中部長に、来週火曜14時のミーティング確認メールを書いて。丁寧だけど簡潔に。件名も付けて。

→ 相手・用件・トーン・形式が明確。すぐ使えるメールが出てくる。

例2: 企画を考える

NG PROMPT
イベントの企画を考えて

→ 業界は？ターゲットは？予算は？目的は？ ぼんやりした企画が量産される。

VS
GOOD PROMPT
30代の子育て世代向け、予算50万円で、来月開催する食育ワークショップの企画を3案出して。各案に「コンセプト」「タイムテーブル」「必要な準備物」を含めて。

→ 条件が具体的なので、すぐ検討に入れる企画が出てくる。

例3: 文章を要約する

NG PROMPT
この文章をまとめて

→ どのくらいの長さ？誰向け？何を重視？ 冗長な要約が返ってくる。

VS
GOOD PROMPT
この議事録を、上司に報告するために3行で要約して。決定事項と次のアクションを必ず含めて。

→ 用途・長さ・必須項目が明確。報告にそのまま使える。

例4: アドバイスを求める

NG PROMPT
売上を上げるにはどうしたらいい？

→ 業種も規模も課題も不明。教科書のような一般論しか返ってこない。

VS
GOOD PROMPT
EC通販で健康食品を販売している。月商500万。新規獲得が頭打ちで、CPAが1万円を超えている。広告費を増やさずにCPAを下げる施策を5つ提案して。

→ 背景情報と具体的な課題を伝えるから、実行可能な施策が返ってくる。

共通パターンが見えましたか？ Good Prompt には必ず「誰に・何を・どのように・どんな形式で」が含まれています。これが基本の考え方です。次のPartで、具体的なテクニックを5つ学びましょう。
02

基本の5つのテクニック

Anthropic公式のベストプラクティスに基づく実践テクニック

テクニック 1: 具体的に書く

最も重要で、最も効果が大きいテクニックです。曖昧な言葉を、具体的な条件に置き換えるだけで、AIの回答精度が劇的に上がります。

BEFORE — 曖昧
いい感じのブログ記事を書いて
↓
AFTER — 具体的
以下の条件でブログ記事を書いて。

- テーマ: 在宅ワークの生産性向上
- ターゲット: 30代の会社員
- 文字数: 1500文字程度
- トーン: フレンドリーだけど信頼感がある
- 構成: 導入 → 課題3つ → 解決策3つ → まとめ
- SEOキーワード:「在宅ワーク 集中」を自然に含める
効果: 「何を・誰に・どのくらい・どんな雰囲気で・どんな構造で」を明示するだけで、AIは迷わず高品質な記事を生成できる。
具体化のチェックリスト
  • 誰に？ — ターゲット・読者・相手
  • 何を？ — テーマ・目的・ゴール
  • どのくらい？ — 文字数・個数・深さ
  • どんなトーン？ — カジュアル・フォーマル・専門的
  • どんな形式？ — 箇条書き・表・見出し付き

テクニック 2: 役割を指定する（ロール設定）

AIに「あなたは〇〇の専門家です」と伝えると、その分野に特化した回答が返ってきます。人間でも「マーケターとして意見を聞かせて」と言われれば、マーケ視点で答えますよね。AIも同じです。

ロール設定なし
この商品の改善点を教えて
→ 一般的で浅い意見が返ってくる
↓
ロール設定あり
あなたは15年の経験を持つマーケティングコンサルタントです。
EC通販の健康食品を扱うクライアントから相談を受けました。

この商品ページを見て、コンバージョン率を上げるための改善点をプロの視点で5つ指摘してください。
→ 専門的で具体的な改善提案が返ってくる。「ファーストビューのCTAが弱い」「社会的証明が不足」など、プロ視点のアドバイスに変わる。

使える役割（ロール）の例

💼
経営コンサルタント

事業戦略・経営課題の分析。「売上構造を分析して」「この事業計画の穴を指摘して」

✍️
プロの編集者

文章の改善・校正。「この原稿をわかりやすくリライトして」「論理の飛躍を指摘して」

📈
データアナリスト

数値分析・レポート作成。「このデータから傾向を読み取って」「異常値を見つけて」

🎓
やさしい先生

わかりやすい解説。「初心者にもわかるように、例え話を使って教えて」

テクニック 3: 出力形式を指定する

AIに「どんな形で答えてほしいか」を伝えると、そのまま使えるアウトプットが得られます。形式を指定しないと、AIが勝手に判断するため、使いにくい出力になりがちです。

形式指定の例
箇条書き
このサービスのメリット・デメリットを箇条書きで5つずつ挙げて
表形式
競合3社の価格・機能・サポート体制を表形式で比較して
見出し付き構成
以下の構成で企画書をまとめて：
1. 背景と課題
2. 提案内容
3. 期待効果
4. スケジュール
5. 予算
ステップ形式
この業務の手順を、新人でも迷わないように番号付きのステップで書いて
よく使う形式指定ワード
  • 「箇条書きで」「番号付きリストで」
  • 「表形式で」「比較表にして」
  • 「〇〇文字以内で」「3行で」
  • 「メリット/デメリットに分けて」
  • 「結論を先に、理由を後に」
  • 「見出しと本文で構成して」

テクニック 4: 例を示す（Few-shot）

「こういう感じで答えてほしい」というサンプルを1〜2個見せるだけで、AIはそのパターンを学習して再現します。これを「Few-shot プロンプティング」と呼びます。

例: 商品レビューの要約
サンプルを見せる
以下の形式で、各レビューを1行に要約してください。

【入力例】
「この化粧水を使い始めて3ヶ月。最初は効果がわからなかったけど、最近肌のキメが整ってきた気がする。ただ値段がちょっと高い。」

【出力例】
★ 肌のキメ改善を実感（3ヶ月使用）/ △ 価格がネック

---
では、以下のレビューを同じ形式で要約してください：
（レビュー本文を貼り付け）
効果: AIは例の「★/△の記号」「ポジ/ネガの分類」「1行の長さ」をすべて学習し、同じフォーマットで出力してくれる。
サンプル入力を見せる
→
サンプル出力を見せる
→
AIがパターンを学習
→
本番データに適用
ポイント サンプルは1〜2個で十分。多すぎると逆にプロンプトが長くなりすぎます。「こういうフォーマットで」「このトーンで」を伝えたいときに最適なテクニックです。

テクニック 5: 段階的に考えさせる（Chain of Thought）

複雑な問題をAIに投げるとき、「ステップバイステップで考えて」と一言添えるだけで、AIの思考が深くなり、回答の精度が劇的に上がります。

一気に答えさせる（精度が低い）
うちのECサイトの売上改善策を教えて
→ 表面的な改善策のリストが返ってくるだけ
↓
段階的に考えさせる（精度が高い）
うちのECサイトの売上改善を考えたい。以下のステップで分析してください。

ステップ1: まず、EC通販の売上は「訪問数 x CVR x 客単価」に分解できます。それぞれの改善施策を挙げて。
ステップ2: 各施策の「効果の大きさ」と「実行の難易度」を評価して。
ステップ3: 効果が大きくて実行が簡単なものTOP3を推薦して、その理由を説明して。
→ 構造的に分析された、優先順位付きの改善提案が返ってくる。
使い方のコツ
  • 「ステップバイステップで考えて」と書くだけでもOK
  • より精度を上げたいなら、ステップの内容まで指定する
  • 特に「比較」「分析」「意思決定」系の質問に効果大
  • AIが途中で間違った場合も、どのステップで間違えたか特定しやすい

5つのテクニック まとめ

# テクニック ひとこと 効果
1 具体的に書く 5W1Hを明示する 効果 MAX
2 役割を指定する 「あなたは〇〇です」 専門性 UP
3 出力形式を指定 表・箇条書き・構成 即使える
4 例を示す サンプルを1〜2個 再現性 UP
5 段階的に考えさせる ステップ分解 深い分析
03

澤田が実際に使っている型

現場で本当に使い倒しているプロンプトパターン5選

3-a. 5つのプロンプトパターン

テクニックを知っても、実際にどう書けばいいかイメージが湧かない方のために、僕が日常的に使っているプロンプトの「型」を5つ紹介します。

1
壁打ち型

アイデアや方針に穴がないか、AIに壁打ち相手になってもらう。

こういう課題があって、こう考えてるんだけど、穴があれば教えて。

【課題】新商品のLPを作るが、ターゲットが40代女性と60代女性で2層いる
【自分の考え】LPを2本作り分けるべきだと思っている

この方針の弱点やリスクがあれば指摘してほしい。別のアプローチがあればそれも提案して。
2
分析型

大量の情報を構造的に整理してもらう。議事録やヒアリングデータに最適。

以下の会議メモを読み込んで、次の3つに分類して。

(1) 決定事項（確定したこと）
(2) 未解決課題（持ち越しになったこと）
(3) ネクストアクション（誰が・いつまでに・何をするか）

---
（会議メモを貼り付け）
3
クリエイティブ型

コピーやアイデアを大量生成させる。切り口の多様性がポイント。

ターゲットは60代女性。健康意識が高く、品質にこだわる層。

この和漢サプリメントのキャッチコピーを10案出して。
条件：
- それぞれ違う切り口で（恐怖訴求・共感・権威・好奇心・実績など）
- 20文字以内
- 押し売り感のない、品のある表現で
4
レビュー型

自分が書いた文章やプランをAIにレビューさせる。観点を指定するのがコツ。

以下の提案書をレビューして。次の3つの観点で指摘してほしい。

(1) 論理の飛躍 — 根拠が弱い箇所
(2) 不足している情報 — 読者が「なぜ？」と思いそうな箇所
(3) 構成の問題 — 順番を入れ替えた方がわかりやすい箇所

---
（提案書を貼り付け）
5
教師型

自分が知らない分野を学ぶときに、AIを「やさしい先生」にする。

私は財務会計の初心者です。

「キャッシュフロー計算書の読み方」について、以下の条件で教えてください。
- 専門用語を使わない（使う場合は必ず日本語で説明を付ける）
- 身近な例え話を使う
- 「なぜそれが重要なのか」を最初に教える
- 最後に理解度チェックの質問を3つ出して
04

やってしまいがちなNG集

初心者がハマりやすい4つの落とし穴

4-a. よくある失敗パターン

プロンプトのテクニックを覚えても、以下の落とし穴にハマると効果が半減します。自分に当てはまるものがないか、チェックしてみてください。

NG 1
長すぎて要点がぼやける

背景情報を書くのは良いことですが、関係ない情報まで詰め込むとAIが「何が一番大事か」を判断できなくなります。「この情報は回答に必要か？」と自問してから書きましょう。

NG 2
一度に複数の無関係な質問をする

「来週の天気と、あと売上レポートも作って、それからメールも書いて」のように、関連性のない質問を一度にまとめると、すべての回答が中途半端になります。1プロンプト = 1テーマが鉄則。

NG 3
前提条件を省略する

「〇〇について教えて」と聞くとき、自分の知識レベル・目的・背景を省略しがち。AIはあなたのことを何も知りません。「私は〇〇の初心者で、△△のために知りたい」と前提を伝えるだけで回答の質が変わります。

NG 4
AIの回答をそのまま使う

AIの回答は「素材」であり「完成品」ではありません。特にビジネス文書や対外的な発信では、必ず自分の目で確認し、判断を加えてから使ってください。「AIが言ったから正しい」は危険です。

覚えておくべき大原則 AIは「超優秀なアシスタント」であって「判断者」ではない。
最終判断は必ず自分で行う。AIはあくまで思考を加速するツール。
