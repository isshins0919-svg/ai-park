NEXT STEPS

航海を終えた後にやること

✅ 最初の1週間

一気にやらなくていい。1日1つずつ

  • CLAUDE.mdを実際のプロジェクトに配置して、Claude Codeを起動してみる
  • identity.mdを書いて、AIの応答トーンを確認する
  • Island 04で作ったSkillを /skill名 で実行してみる
  • MCPで繋がっているツール（Slack, Calendar, Drive等）を確認する
  • Nice Diveを1回書いて、学びの蓄積を始める

🗺️ 最初の1ヶ月

環境を育てていく

  • Skillを3つ以上作って、日常業務に組み込む
  • サブエージェント（agents/）を1体作って、Skillから呼び出してみる
  • knowledge/ にN1定義やブランドガイドを追加する
  • rules/ にセキュリティルールを追加する
  • Nice Diveログを週1で振り返り、繰り返しのミスをルール化する

🏁 最終チェックポイント

  • CLAUDE.md（Tier 0設計）が完成している
  • identity.md（AI人格）が定義されている
  • Skill（commands/）が1つ以上ある
  • エージェント組織図（commands/ + agents/）が設計されている
  • ルール（rules/）が1つ以上ある
  • Nice Diveログが1件以上記録されている

ゴール

Claude Codeで動く、自分の業務用AIエージェント環境一式を完成させる

  1. フォルダ構造を作る — Claude Codeで以下を実行:
    以下のフォルダ構造を作成してください:
    .claude/commands/
    .claude/agents/
    .claude/knowledge/
    .claude/rules/
    .claude/logs/
    .claude/handoff/
  2. CLAUDE.mdを配置 — Island 06で作ったCLAUDE.mdをプロジェクトルートに置く
  3. identity.mdを作る — Claude Codeで以下を実行:
    starter-kit/identity-template.md を読んで、
    以下の設定で identity.md を作成してください。

    # 人格設定
    - 名前: [AIの名前を自由に]
    - トーン: [例: 丁寧だけど簡潔、関西弁、etc.]
    - 口癖: [例: 「了解です！」]
    - やらないこと: [例: 曖昧な返事をしない]
  4. Skillを配置 — Island 04で作ったSkillを .claude/commands/ に置く
  5. ルールを配置 — starter-kit の security.md と anonymize.md を .claude/rules/ に配置
  6. 動作確認 — Claude Codeを起動して、CLAUDE.mdが正しく読み込まれるか確認
  7. Nice Diveを書く — この航海全体の振り返りをNice Diveフォーマットで記録
所要時間: 約45分
