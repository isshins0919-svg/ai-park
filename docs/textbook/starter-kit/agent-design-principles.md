# エージェント設計の基本原則

## 7つのルール

1. **目的・ゴール・行動指針・制約条件を必ず持たせる**
   - 人間の組織と同じ。役割が曖昧だと暴走する

2. **1エージェント1タスク**
   - 複数タスクを持たせるとコンテキストが溢れて品質が落ちる

3. **まず1体動かしてから組織を拡大**
   - 最初から大組織を作らない。1体が安定してから次を作る

4. **新エージェントは上司エージェントにレビューさせる**
   - 人間がレビューするより、関連エージェントにレビューさせた方が品質が高い

5. **自走ルールを渡す**
   - 「3回自分で試してから聞く」等のルールで、不要な確認を減らす

6. **制約は後入れ**
   - 最初は自由に作らせる → 後から法規制等のチェックを入れる
   - 最初から制約をかけると、守りに入って良いものが出ない

7. **エンベディング（ベクトル化）なしで良いクリエイティブは作れない**
   - 200本のデータがあれば十分な精度が出る
   - データの質（売れた/売れてないのラベル）が量より重要

## agents/ vs commands/ の使い分け

| | commands/ | agents/ |
|---|---|---|
| 起動方法 | ユーザーが `/skill` で手動 | AIが Agent tool で自動 |
| コンテキスト | 親と共有（Tier 2） | 独立空間（Tier 3） |
| 向いてる用途 | 対話型・オーケストレーター | レビュー・評価・並列タスク |

**判断基準**: ユーザーとの対話が必要 → commands/ / 独立して完結する専門タスク → agents/

## エージェント分割の判断基準

以下の3つのNG信号が出たら、タスクを分割する:

1. **ループ3回以上NG** — 同じミスを3回繰り返す → 複雑すぎる
2. **指示忘れ** — 最初の条件が途中から無視される → CW圧迫
3. **ハルシネーション** — 存在しない情報を言い始める → 脳がパンク
