WHAT IS MCP

MCP — AIが「考えるだけ」から「実行できる」に変わる仕組み

MCP（Model Context Protocol）は、AIが外部サービスを操作するための統一規格。
Anthropicが2024年に策定したオープンプロトコルで、どのAIモデルでも同じ方法で外部ツールに繋げられます。

🔌
MCPを一言で言うと
「AIのためのUSB」
USB以前は、プリンターもマウスもそれぞれ違う端子だった。
USBで統一されて「何でも刺せば使える」になった。
MCPも同じ。Slack・Google・Canva…全部同じ規格でAIに繋がる。

🏗️ MCPの3つの登場人物

ホスト・クライアント・サーバー — レストランで例えると分かりやすい

MCPの構造

 お客さん       ウェイター      厨房
🍽️ レストランの例

ホスト（お客さん） = Claude Code。「Slackに投稿して」とリクエストする

クライアント（ウェイター） = Claude Codeに内蔵。リクエストを適切なサーバーに届ける

サーバー（厨房） = Slack MCP。実際にSlack APIを叩いて投稿する

※ サーバーとは「裏側で動いて、リクエストが来たら仕事をこなす縁の下のプログラム」のこと。物理的なマシンじゃなくて、裏方スタッフをイメージして。

⚓ ポイント

ユーザーはホストとサーバーだけ意識すればOK。

クライアント（ウェイター）はClaude Codeが自動でやってくれるので、設定するのは「どのサーバー（厨房）を繋ぐか」だけ。

SERVER TYPES

MCPサーバーの2つの種類

MCPサーバーには「ローカル」と「リモート」の2種類があります。

💻 ローカルサーバー（stdio方式）

自分のPCで動くサーバー。セットアップが必要だが自由度が高い

自分のPCにインストールして使うタイプ。npm（エヌピーエム。JavaScript用のアプリ配布所。スマホのApp Storeみたいなもの）やpip（ピップ。Python用のApp Store）でパッケージ（機能をまとめた部品ファイル）を入れて、settings.json（Claude Codeの設定ファイル。「どのサーバーを繋ぐか」を書くメモ帳みたいなもの）に設定を書きます。

// settings.json の設定例（ファイルシステムサーバー）
{
 "mcpServers": {
  "filesystem": {
   "command": "npx",
   "args": ["@anthropic/mcp-fs", "/Users/me/Documents"]
  }
 }
}
メリット / デメリット

✅ 自由度が高い（カスタムサーバーも作れる）
✅ データが外部に出ない（セキュリティ◎）
❌ 環境構築（=アプリを動かす土台を準備すること）が必要。Node.js（ノード。JavaScriptをPCで動かす土台）や Python（パイソン。プログラミング言語の一つ。AI・データ分析で定番）のインストールが前提
❌ PCが起動していないと使えない

☁️ リモートサーバー（OAuth方式）

ワンクリックで繋がる。設定不要だが提供されているものだけ

Claude Codeの設定画面やclaude.ai上で「Connect」するだけで繋がるタイプ。Google Calendar、Slack、Canvaなど大手サービスはこちら。

メリット / デメリット

✅ 設定が超簡単（ワンクリック）
✅ 常時接続（PCが落ちてても動く）
❌ 提供されているサービスに限られる
❌ データが外部サーバーを経由する

HOW TO SETUP

MCPサーバーの繋ぎ方

1️⃣ リモートサーバーの場合（簡単）

1

Claude Codeのチャットで聞く

「Google Calendarを繋ぎたい」と伝えるだけで、Claude Codeが接続ウィザードを案内してくれます。

2

OAuthで認証する

ブラウザが開くので、Googleアカウントでログインして権限を許可する。

3

使い始める

「今日の予定は？」と聞くだけで、カレンダーから予定を取得してくれる。

2️⃣ ローカルサーバーの場合

1

CLIで追加する

CLI（シーエルアイ / Command Line Interface = ターミナルに文字を打ってPCを操作する画面）で設定を追加します。

claude mcp add サーバー名 -- コマンド 引数

※ コマンド = PCへの命令（例: npx）、引数 = 命令にわたす追加情報（例: ファイル名やフォルダのパス）

2

設定を確認する

claude mcp list で、現在繋がっているサーバーの一覧を確認。

3

Claude Codeを再起動

設定変更後はClaude Codeを再起動すると反映されます。

# MCPサーバーの追加コマンド例

# ファイルシステムサーバーを追加
claude mcp add filesystem -- npx @anthropic/mcp-fs /Users/me/Documents

# 現在の設定を確認
claude mcp list

# サーバーを削除
claude mcp remove filesystem
SAFETY

MCPを使う上での注意点

🛡️ 権限管理 — AIに何を許可するか

MCPは強力だからこそ、安全装置が必要

MCPでSlackに接続すると、AIは本当にメッセージを送信できます。つまり間違った相手に間違ったメッセージを送る可能性がある。

❌ 危ない使い方

・全権限を無条件で許可する
・MCPサーバーの出所を確認しない
・重要な操作をAIに丸投げする
・APIキーをコードに直書きする

✅ 安全な使い方

・操作ごとにAIに確認させる（デフォルト設定）
・信頼できるサーバーのみ使用する
・重要な操作は人間がレビューする
・APIキーは環境変数で管理する

Claude Codeの安全設計

Claude Codeはデフォルトで操作のたびにユーザーに確認を求めます（「Slackにメッセージを送っていいですか？」等）。慣れてきたら、信頼できる操作だけ自動承認に切り替えられます。最初はデフォルト（都度確認）のままが安全。

🏁 チェックポイント — この島で覚えること

  • MCPは「AIのためのUSB」。外部サービスへの統一的な接続規格
  • ローカル（自分のPC上）とリモート（クラウド経由）の2種類がある
  • 権限管理が重要。最初はデフォルト（都度確認）で運用し、慣れてから自動化する

📖 この島の航海用語

MCP（Model Context Protocol）

AIが外部ツールに接続するための標準規格。「AIのUSB」。

MCPホスト

MCPの管理者。Claude Code自体がホスト。

MCPクライアント

ホストとサーバーを中継する仲介役。

MCPサーバー

実際にツールを提供する側。Slack、Google Drive、DB等。

ローカルMCP

自分のPC上で動くMCPサーバー。ファイル操作やDB接続向き。

リモートMCP

クラウド上で動くMCPサーバー。Slack、Google Calendar等の外部サービス。

サーバー

裏側で動いて、リクエストが来たら仕事をこなす縁の下のプログラム。物理的な機械じゃなく「裏方スタッフ」のイメージ。

CLI（コマンドラインインターフェース）

ターミナルに文字を打ってPCを操作する画面。GUI（マウスで操作）の反対。Claude Codeの本体はここで動く。

コマンド / 引数

コマンド＝PCへの命令（例: npx）、引数＝命令にわたす追加情報（例: ファイル名やフォルダのパス）。

npm / pip

npm=JavaScript用、pip=Python用の「アプリ配布所」。スマホのApp Store的なもので、ここから機能を追加できる。

パッケージ

機能をまとめたソフトウェアの部品ファイル。これを入れると新しい機能が使えるようになる。

settings.json

Claude Codeの設定を書くファイル。「どのMCPサーバーを繋ぐか」などをここに書く。拡張子 .json は「設定をキレイに並べるフォーマット」。

環境構築

アプリを動かすのに必要な土台（言語や関連ツール）をPCに用意すること。家を建てる前の「基礎工事」のようなもの。

Node.js / Python

Node.js=JavaScriptをPCで動かす土台。Python=プログラミング言語の一つで、AI・データ分析で広く使われる。どちらもMCPサーバーを動かす土台になる。
