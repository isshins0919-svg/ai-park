WHAT IS MCP

MCP — AIが「考えるだけ」から「実行できる」に変わる仕組み

MCP（Model Context Protocol）は、AIが外部サービスを操作するための統一規格。
Anthropicが2024年に策定したオープンプロトコルで、どのAIモデルでも同じ方法で外部ツールに繋げられます。

🔌
MCPを一言で言うと
「AIのためのUSB」
USB以前は、プリンターもマウスもそれぞれ違う端子だった。
USBで統一されて「何でも刺せば使える」になった。
MCPも同じ。Slack・Google・Canva…全部同じ規格でAIに繋がる。

🏗️ MCPの3つの登場人物

ホスト・クライアント・サーバー — レストランで例えると分かりやすい

MCPの構造

┌─────────────┐   ┌──────────────┐   ┌──────────────┐
│ MCP ホスト │ ←→ │ MCP クライアント│ ←→ │ MCP サーバー │
│ Claude Code │   │ （内蔵）    │   │ Slack, Google │
└─────────────┘   └──────────────┘   └──────────────┘

 お客さん       ウェイター      厨房
🍽️ レストランの例

ホスト（お客さん） = Claude Code。「Slackに投稿して」とリクエストする

クライアント（ウェイター） = Claude Codeに内蔵。リクエストを適切なサーバーに届ける

サーバー（厨房） = Slack MCP。実際にSlack APIを叩いて投稿する

⚓ ポイント

ユーザーはホストとサーバーだけ意識すればOK。

クライアント（ウェイター）はClaude Codeが自動でやってくれるので、設定するのは「どのサーバー（厨房）を繋ぐか」だけ。

SERVER TYPES

MCPサーバーの2つの種類

MCPサーバーには「ローカル」と「リモート」の2種類があります。

💻 ローカルサーバー（stdio方式）

自分のPCで動くサーバー。セットアップが必要だが自由度が高い

自分のPCにインストールして使うタイプ。npmやpipでパッケージを入れて、settings.jsonに設定を書きます。

// settings.json の設定例（ファイルシステムサーバー）
{
 "mcpServers": {
  "filesystem": {
   "command": "npx",
   "args": ["@anthropic/mcp-fs", "/Users/me/Documents"]
  }
 }
}
メリット / デメリット

✅ 自由度が高い（カスタムサーバーも作れる）
✅ データが外部に出ない（セキュリティ◎）
❌ 環境構築が必要（Node.js, Python等）
❌ PCが起動していないと使えない

☁️ リモートサーバー（OAuth方式）

ワンクリックで繋がる。設定不要だが提供されているものだけ

Claude Codeの設定画面やclaude.ai上で「Connect」するだけで繋がるタイプ。Google Calendar、Slack、Canvaなど大手サービスはこちら。

メリット / デメリット

✅ 設定が超簡単（ワンクリック）
✅ 常時接続（PCが落ちてても動く）
❌ 提供されているサービスに限られる
❌ データが外部サーバーを経由する

HOW TO SETUP

MCPサーバーの繋ぎ方

1️⃣ リモートサーバーの場合（簡単）

1

Claude Codeのチャットで聞く

「Google Calendarを繋ぎたい」と伝えるだけで、Claude Codeが接続ウィザードを案内してくれます。

2

OAuthで認証する

ブラウザが開くので、Googleアカウントでログインして権限を許可する。

3

使い始める

「今日の予定は？」と聞くだけで、カレンダーから予定を取得してくれる。

2️⃣ ローカルサーバーの場合

1

CLIで追加する

claude mcp add サーバー名 -- コマンド 引数 で設定を追加します。

2

設定を確認する

claude mcp list で、現在繋がっているサーバーの一覧を確認。

3

Claude Codeを再起動

設定変更後はClaude Codeを再起動すると反映されます。

# MCPサーバーの追加コマンド例

# ファイルシステムサーバーを追加
claude mcp add filesystem -- npx @anthropic/mcp-fs /Users/me/Documents

# 現在の設定を確認
claude mcp list

# サーバーを削除
claude mcp remove filesystem
SAFETY

MCPを使う上での注意点

🛡️ 権限管理 — AIに何を許可するか

MCPは強力だからこそ、安全装置が必要

MCPでSlackに接続すると、AIは本当にメッセージを送信できます。つまり間違った相手に間違ったメッセージを送る可能性がある。

❌ 危ない使い方

・全権限を無条件で許可する
・MCPサーバーの出所を確認しない
・重要な操作をAIに丸投げする
・APIキーをコードに直書きする

✅ 安全な使い方

・操作ごとにAIに確認させる（デフォルト設定）
・信頼できるサーバーのみ使用する
・重要な操作は人間がレビューする
・APIキーは環境変数で管理する

Claude Codeの安全設計

Claude Codeはデフォルトで操作のたびにユーザーに確認を求めます（「Slackにメッセージを送っていいですか？」等）。慣れてきたら、信頼できる操作だけ自動承認に切り替えられます。最初はデフォルト（都度確認）のままが安全。

🏁 チェックポイント — この島で覚えること

  • MCPは「AIのためのUSB」。外部サービスへの統一的な接続規格
  • ローカル（自分のPC上）とリモート（クラウド経由）の2種類がある
  • 権限管理が重要。最初はデフォルト（都度確認）で運用し、慣れてから自動化する

🔍 港の検査 — MCP接続を棚卸しする

自分のClaude Codeに何が繋がっているか確認し、足りないものを見つけるハンズオン。
「今のAIに何ができて、何ができないか」を可視化する大事なステップです。
