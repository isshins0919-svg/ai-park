AIへの指示の技術

「良い指示」で、AIの出力は10倍変わる

本編 15分 + ミッション 15分

THE 5 ELEMENTS

AIへの指示の5要素

AIへの指示は「過不足なく」書くのが理想。多すぎると読まれない。少なすぎると誤解される。
判断基準はシンプル: 「これを書かなかったら、AIが間違った方向に進むか？」

指示の5要素テンプレート

① 役割: あなたは [○○の専門家] です
② 目的: [○○] を達成してください
③ 背景: この案件は [○○] の文脈で [○○] な対象向けです
④ 制約: [○○] はしないでください / [○○] の範囲内で
⑤ 参考: [○○.md] を読んでから始めてください

この5つが揃っていれば、AIは迷わず動けます。逆に1つでも欠けると「いい感じにやっといて」と同じ結果になります。

🎭 ① 役割 — 「誰として」考えるか

同じ質問でも、専門家と一般人では答えが変わる

AIに「あなたは広告コピーライターです」と伝えるだけで、出力の質が変わります。役割を指定すると、AIはその分野の専門知識と思考パターンを優先的に使います。

❌ 役割なし

「膝サポーターのキャッチコピーを書いて」

→ 汎用的で当たり障りのないコピーが出る

✅ 役割あり

「あなたはシニア向け健康商品の広告コピーライターです。膝サポーターのキャッチコピーを書いて」

→ ターゲットの痛みに寄り添ったコピーが出る

🎯 ② 目的 — 「何を達成する」か

アウトプットではなく、アウトカムを伝える

「バナーを作って」ではなく「クリック率を上げるバナーを作って」。最終的に何を達成したいかを伝えると、AIの判断基準が明確になります。

なぜ大事？

「報告書を作って」= アウトプット（成果物）を指示。
「経営陣が投資判断できる報告書を作って」= アウトカム（結果）を指示。
後者の方が、AIは何を強調し何を省くかを判断できる。

🌍 ③ 背景 — 「なぜこれをやる」のか

文脈があるから、的外れを防げる

同じ「メール文面を書いて」でも、謝罪メールと営業メールでは全く違います。背景を伝えることで、AIはTPOに合った出力を返せます。

🍽️ レストランの例

「おすすめは？」だけだと、ウェイターは困る。
「今日は記念日なんです。2人で、予算1万円くらいで」と言えば、ぴったりのコースを提案してくれる。
背景 = ウェイターへの「空気読んで」の代わり。

⚓ 実例

「この案件はシニア女性向けの膝サポーター。競合は○○で、うちの強みは医師推薦。初回半額キャンペーンで新規獲得を狙っている」

これだけの背景があれば、AIは競合との差別化を意識したコピーを書ける。

🚫 ④ 制約 — 「やってはいけない」こと

自由すぎるAIは暴走する。柵を作る

AIは指示されないと何でもやります。薬機法に触れるコピーも、3000文字の長文も、英語の回答も。制約は「やらない」を明確にする安全装置です。

🚗 車線の例

道路に白線がなかったら？ → 全員が好き勝手に走って事故る。
白線 = 制約。自由を少し制限することで、全体が安全に動く。
制約は「縛り」ではなく「道しるべ」。

⚓ よく使う制約

・薬機法に触れる表現は使わない
・1文は40文字以内
・HTMLで出力（Markdown不可）
・○○.md のトーンに合わせる
・個人情報は匿名化する

📚 ⑤ 参考 — 「これを読んでから」動く

ナレッジを渡せば、素人が一瞬で専門家になる

AIに専門知識がなくても、参考資料を渡せば学習できます。Claude Codeでは .claude/knowledge/ や .claude/clients/ にナレッジファイルを置いておき、指示で「これを読んで」と指定します。

なぜ大事？

新入社員に「先輩のレポート3本読んでから書いて」と言えば、いきなり書くより格段に良いものが出る。AIも全く同じ。参考を渡す = 即席の専門教育。

SHOT DESIGN

例を見せる技術 — 「Shot」で精度を爆上げ

5要素テンプレートで「何をするか」を伝えたら、次は「どんなアウトプットが正解か」を具体例（Shot）で見せます。ポイントは「数より多様性」。

🎯 3つのShotで90%カバー

10個の似た例より、3つの多様な例

1

正常系の例（典型的なケース）

「こういう入力が来たら、こういう出力を出す」の王道パターン。料理で言えば「完成写真」。

2

異常系の例（NGケース）

「こういうダメな出力は出さない」のパターン。料理で言えば「焦がした写真」。

3

境界ケースの例（判断に迷うケース）

「微妙な判断が求められるとき、こう判断する」のパターン。料理で言えば「ギリギリOKの焼き色」。

👨‍🍳 料理の例

レシピに「完成写真」が1枚あるだけで、文章だけのレシピより格段にわかりやすい。さらに「失敗例」と「ギリギリOK」の写真があれば、判断基準がクリアになる。

⚓ 実例

バナー制作Skillには:
✅ 良い例（N1に刺さるコピー）
❌ NG例（機能説明だけ）
🔶 微妙な例（方向性は合ってるけど弱い）

3パターンあるから「何が良くて何がダメか」を判断できる。

PRO TIPS

すぐ使える3つのテクニック

🔗 テクニック1: 「段階的に」指示する

一度に全部頼まず、ステップを刻む

❌ 一括指示

「競合を分析して、コンセプトを決めて、コピーを10本書いて、バナーも作って」

→ 全部が中途半端になる。CWも一気に埋まる

✅ 段階指示

Step 1:「まず競合3社を分析して」
Step 2:「分析結果をもとにコンセプトを3案出して」
Step 3:「コンセプトAでコピーを10本書いて」

→ 各ステップで確認・修正ができる

🪞 テクニック2: 「考え方を見せて」と頼む

答えだけでなく、思考プロセスを出力させる

「答えを出す前に、まず考え方を箇条書きで見せて」と追加するだけで、AIの思考が可視化されます。間違った前提で突き進むのを防げます。

# 考え方を見せてもらう指示の例
このコピーを改善してください。

ただし改善案を出す前に、まず:
1. 現状のコピーの問題点を3つ挙げて
2. ターゲットが本当に求めていることを推測して
3. その上で改善の方向性を提示して

それから改善案を書いてください。

🔄 テクニック3: 「ダメ出し」を前提にする

最初から完璧を求めず、キャッチボールで磨く

AIの一発目の出力は「叩き台」。そこからフィードバックを繰り返して磨いていくのが、最も品質が高くなるアプローチです。

ポイント

ダメ出しの質も大事。「なんか違う」ではなく「なぜ違うか」を伝える。
❌「もっと良くして」→ AIは何を直せばいいかわからない
✅「ターゲットの悩みに寄り添う表現が弱い。もっと具体的な痛みのシーンを入れて」

🏁 チェックポイント — この島で覚えること

  • 指示は5要素（役割・目的・背景・制約・参考）で書く。判断基準は「書かなかったらAIが間違うか？」
  • 例（Shot）は「正常・異常・境界」の3パターンで多様性を確保する
  • 一発で完璧を求めず、段階的に指示して、考え方を可視化し、フィードバックで磨く

🍕 ガレー船の初注文 — 5要素で指示を書く

5要素テンプレートを使って、実際にClaude Codeに仕事を頼むハンズオン。
「曖昧な指示」と「5要素入りの指示」で出力がどう変わるか体験します。

ゴール

同じタスクを「曖昧な指示」と「5要素入りの指示」の両方で試し、違いを体感する

  1. Claude Codeを開く
  2. まず曖昧な指示で試す:
    メールの文面を書いて。お客さんへのお礼メール。
  3. 出力を確認する。汎用的で当たり障りのない内容になっているはず
  4. 次に5要素入りの指示で試す:
    # ① 役割
    あなたはBtoBのSaaS企業の営業担当者です。

    # ② 目的
    初回商談後のお礼メールを書いてください。
    次回のデモ設定に繋げることがゴールです。

    # ③ 背景
    先方は従業員50名のWeb制作会社の部長。
    現在はExcelで案件管理しており、ミスが多いのが悩み。
    商談では当社のプロジェクト管理ツールに興味を示した。

    # ④ 制約
    - 300文字以内
    - 売り込み感を出さない
    - 「お時間をいただきありがとうございました」から始めない（定型すぎる）

    # ⑤ 参考
    トーンは「丁寧だけど親しみやすい」。
    「御社の○○という課題に対して」のように、商談で聞いた具体的な
    悩みに触れると効果的。
  5. 2つの出力を並べて比較する。5要素版の方が、相手に刺さる具体的な内容になっているはず
  6. （余裕があれば）自分の仕事で実際に使う指示を5要素で書いてみる
所要時間: 約15分
成果物

5要素テンプレートで書いた「自分の仕事で使える指示」1つ。これがIsland 04「Skill設計」の土台になります。
