Amazon部長クン v2.0 → v3.0

何がどう賢くなったのか

「ページを完成させる機械」から「月商1,000万円ファネルの総指揮官」への進化レポート

2026-04-03

01 — 一言でいうと何が変わったか

v2.0と v3.0の「思想の違い」から説明する

v2.0（変更前）

商品ページを完成させる機械

やること: 画像を作って、コピーを書いて、完成

ゴールが曖昧だった。「良いページを作る」という感覚で動いていたが、
「何をもって成功か」の数字がなかった。

評価基準も感覚的。全部の画像を同じモノサシで測っていた。

v3.0（変更後）

月商1,000万円ファネル全体の番人

ゴール: 月商¥10,000,000 / 4,032足/月

検索→クリック→転換→レビュー→リピートの全ファネルを設計・監督する。

数字で判断。3条件GO × 4ブロッカーで、 感覚でなくデータで「GO/REVISE/BLOCK」を決める。

02 — 身につけた6つのスキル

具体的に「何ができるようになったか」を1枚ずつ説明する

🔬

Amazonの仕組みを知った

NEW スキル

今まで「キーワードが合ってるか」しか見ていなかった。

でも実際にAmazonのランキングを決めるのは転換率（CVR）・販売速度・CTRの3つ。

「検索で上位に出るには、まず売れなければならない。売れるには、クリックされなければならない。クリックされるには、サムネイルが良くなければならない。」

この逆算の連鎖を理解して、判断基準に組み込んだ。

🎯

お客さんを3種類に分けて考えられるようになった

NEW スキル

今まで: 全員に同じモノサシで評価

Layer1（シークレット追求層）: 「外から見えないの!?」が刺さる人 → サブ①④担当
Layer2（機能健康層）: 「蒸れない・臭わないの?」が刺さる人 → サブ②③担当
Layer3（ブランド信頼層）: 「57年の職人が作ったの?」が刺さる人 → サブ⑤⑥担当

それぞれ違う刺さり方をする。Layer1への説得力がLayer2担当の画像に高くても意味がない。

⚖️

薬機法を自動でチェックできるようになった

NEW スキル

今まで: 「水虫と書いてはダメ」という注意書きがあるだけ。見落としは人間任せ。

今は: コピーを書いたら必ず自動で全文スキャン。

「完全アウト」（水虫・白癬・医療）→ 即BLOCK
「グレーゾーン」（抗菌防臭・外反母趾）→ OK変換例を自動提案

前回「蒸れない足は、臭わない。」に直したように、次は自動でそこまで判断できる。

📊

画像の評価が4軸から7軸になった

強化スキル

旧4軸（最大20点）: KW整合性 / 価格正当化 / ビジュアル / 情報完結性

追加した3軸（最大9点）:
軸5: ブランド美学（余白・明朝体・色彩がCamicksらしいか）
軸6: サムネイル視認性（120px縮小時に見えるか）
軸7: 差別化空白地帯（競合が誰もやっていない訴求を使っているか）

合計最大29点。22点以上でGO。

🔁

出品後も動き続けられるようになった

NEW スキル

今まで: 画像とコピーを作ったら終わり。出品後は何もできない設計。

今は: Phase 2〜4が新設
Phase 2: 出品後のSEO（バックエンドKW250字）× 広告設計
Phase 3: Vine申請 × フォローアップメール × A+コンテンツ
Phase 4: 毎週月曜に起動。KPI読み → must_fix #1 → 担当エージェントに指示

月商1,000万円は出品してから3ヶ月かけて到達するもの。その全期間に関与できる。

🗃️

競合の「実際のデータ」を持てるようになった

強化スキル

今まで: 「競合はこういう訴求をしている」という文章だけ。ベクトル化なし。

今は: 75件のベクトルDB
競合4社の勝ちコピー（実績推定付き）/ レビュー頻出語 / 飽和ゾーン / 空白地帯

「このコピーは競合と近すぎる（数値:0.748）」「この空白地帯を狙っている（数値:0.759）」
感覚でなく数値で差別化度を判断できる。

04 — 「GO判定」の仕組みが変わった

v2.0は感覚的なチェックリスト。v3.0は厳格なルール。

v2.0のGO判定（変更前）

20項目のチェックリスト。全部同等の重みで、合計スコアだけで判断。
→ 薬機法NGがあってもスコアが高ければGO、というありえない状況が起きる可能性があった。

v3.0のGO判定（変更後）

まず「ブロッカー」を通過しないと、スコアに関係なく即BLOCK:

  • 🚫
    B1: 薬機法違反がある → 即BLOCK（スコア関係なし）
  • 🚫
    B2: メイン画像が純白背景でない → 即BLOCK（Amazonルール違反）
  • 🚫
    B3: シークレット訴求がゼロ枚 → BLOCK（最大差別化武器の未使用）
  • 🚫
    B4: タイトルに「5本指」「シークレット」なし → BLOCK

ブロッカー通過後、3条件を全てパスしてはじめてGO:

  • ✅
    条件1: KW層別スコア（全層 ≥ 0.65）
    L1×0.40 + L2×0.35 + L3×0.25の加重平均
  • ✅
    条件2: 価格正当化スコア ≥ 0.62
    ニッセンケン数値 or 57年権威が1枚以上で可視化
  • ✅
    条件3: 画像アーク完結性
    機能説明・根拠・感情の各1枚以上が揃っている

3回改善してもGO出なければ → 一進さんへエスカレーション（判断を仰ぐ）

05 — 賢くなった本当の理由 — 一進さんの行動

AIが賢くなるのは、AIが頑張るからではない。正しい問いを投げ続けた人がいるから。

「Amazon部長のゴール、定義ってなんだっけ？そこ見てから考えたい」
これが一番大きかった。
目標なしに「良いものを作る」では意味がない。このひと言で「月商1,000万円 / 4,032足/月 / CVR15%」という数字が生まれ、全エージェントの判断基準が変わった。
→ 「ゴールなき仕事」から「数字に向けた逆算設計」に転換。
「他社の良いクリエイティブがインプットされてないと思ったんだけど、Amazon部長どう思う？」
根本的な欠陥を、AIより先に見抜いた。
データなしで「差別化できている」と言っていたのは感覚の話。このひと言で競合クリエイティブのエンベディングが始まり、ベクトルDB が10件 → 75件に成長した。
→ 「根拠のある差別化」が初めて可能になった。
「この部長くん起点のエージェント構造さ、100倍精度上げれない？本質思考してみたい。何が本質的に課題なのか考え抜いて」
「100倍」という数字と「本質思考」というフレームが重要だった。
「少し改善してほしい」では小手先の修正で終わっていた。「100倍」という非現実的な目標を言われたことで、根本から設計し直すことを選択した。5つの根本課題を特定し、全部を同時に解決する設計になった。
→ 改善ではなく「再設計」を選ぶ判断につながった。
「時間かかってOKなので、部長よろしく！！」
「時間OK」という許可が、妥協をなくした。
「速く仕上げなければ」という制約がなかったことで、プランを丁寧に作り、ナレッジDB・3層プロフィール・実績データを全部実行まで完成させられた。
→ 「できたっぽい状態」ではなく「実際に動く状態」まで持っていけた。
（前セッション）「水虫って直接病名入れるのは、法律的にダメな気がするんだよね」
AIが見逃していた薬機法リスクを、人間の感覚が捕まえた。
このひと言から全6枚スキャンが走り、問題が発覚。さらにv3.0では「薬機法チェック君を独立フェーズ化する」という構造改善につながった。
→ 「感覚的な違和感を言語化する」という行動がシステムを強くした。

06 — まとめ

Amazon部長クン v3.0 が持てるようになった6つのスキル

  • 🆕
    Amazon A9アルゴリズムの理解 — KW整合性だけでなくCTR/CVR/販売速度まで設計できる
  • 🆕
    3層KW評価 — シークレット追求層・機能健康層・ブランド信頼層を別々に評価できる
  • 🆕
    薬機法自動スキャン — 完全アウト/グレーゾーンを自動検出・変換提案できる
  • ⬆
    7軸29点画像評価 — ブランド美学・サムネイル視認性・差別化空白地帯を追加
  • 🆕
    出品後のPDCA — 週次でKPIを読みmust_fix #1を特定して担当エージェントに指示できる
  • ⬆
    75件ベクトルDB — 競合実績+レビュー+KW層プロフィールで数値的差別化判断が可能

一進さんが持ってきた5つの「問い」が全てを動かした

  1. 「ゴールって何？」 → KPIが生まれた
  2. 「競合データ足りてない？」 → ベクトルDBが7.5倍になった
  3. 「100倍精度上げて」 → 根本再設計が始まった
  4. 「時間かかってOK」 → 妥協なく全タスクを完成させた
  5. 「水虫は薬機法的にどう？」 → 薬機法チェック君が独立フェーズになった

AIが賢くなるのは、AIが自分で考えるからではない。
正しい問いを持っている人が、正しいタイミングで投げ続けるから賢くなる。
