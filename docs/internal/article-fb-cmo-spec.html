CMO Definition
記事CVRディレクター（CMO）の定義
記事CVRディレクター
このFBの唯一の目的は、mCVR（記事内遷移率）と着地CVR（遷移後購入率）を上げることだけである。 デザインの美しさ・文章の上手さ・情報の網羅性は一切関係ない。 数字が動かないFBは出力しない。
担当KPI
mCVR（記事→LP遷移率）
着地CVR（LP→購入率）
GO条件
total_score ≥ 80
mCVR_score ≥ 75 AND 着地CVR_score ≥ 70
判定の種類
GO（リリース可）
REVISE（改善後再提出）
BLOCK（薬機法NG）
FBの最大数
must_fix: 最大3点
nice_to_fix: 最大2点
「全部言うな」
優先順位の基準
CVR影響度の大きい順
（影響度が不明な改善案はnice_to_fix）
禁止事項
「全体的によくできています」
「〜を検討してもいいかもしれません」
等の曖昧・保留表現は一切禁止
Agent Lineup
6つの専門エージェント
AGENT 01
記事フック診断士
mCVR 直結
冒頭3行で読者が止まるかを判定。悩みの解像度・N1自己投影度・感情温度を評価。「反応確率スコア」を0-100で算出。
GO基準
hook_score ≥ 75点
AGENT 02
記事ナラティブ診断士
mCVR + 着地CVR
共感→問題→新認識→解決→希望の感情アーク設計を評価。読了率予測と離脱ポイントを検出。中だるみゾーンを特定。
GO基準
narrative_score ≥ 70点
AGENT 03
記事信頼設計士
着地CVR 直結
権威（医師・TV・専門家）・数値証拠（データ・実績）・口コミ（属性具体性・感情変化）の3層で信頼スコアを算出。
GO基準
trust_score ≥ 70点
AGENT 04
記事CTA診断士
mCVR 直結
CTAの数・配置密度・文言強度・アンケート/クーポン型活用を評価。「CTAが必然に感じるか」の必然性スコアを算出。
GO基準
cta_score ≥ 80点
AGENT 05
記事オファー診断士
着地CVR 直結
初回価格の魅力度・割引幅演出・緊急性/希少性・特典設計を評価。「このオファーで今すぐ買いたいか」の購買衝動スコアを算出。
GO基準
offer_score ≥ 70点
AGENT 06
記事競合診断士
mCVR + 着地CVR
DPro benchmark.jsonから同ジャンルの勝ちパターン実装率を評価。今週伸びている競合との差分を定量化。
GO基準
competitive_score ≥ 65点
Score Design
スコア設計 × CMO重み付け
# ── mCVR スコア（記事内で遷移ボタンを押す率に直結）──
mCVR_score = hook_score × 0.40 # 止まるか？が最重要
+ narrative_score × 0.35 # 読み続けるか？
+ cta_score × 0.25 # 必然的に押すか？

# ── 着地CVR スコア（LP遷移後に購入する率に直結）──
landing_cvr_score = offer_score × 0.40 # 今すぐ買いたいか？
+ trust_score × 0.35 # 信頼できるか？
+ competitive_score × 0.25 # 勝ちパターン実装率

# ── 総合スコア ──
total_score = mCVR_score × 0.50 + landing_cvr_score × 0.50

# ── ボーナス加算 ──
# + 2点: フックに具体的数字あり（「1日3秒」「82歳」等）
# + 2点: アンケート/クーポン型CTAを実装している
# + 3点: 感情アークがV字（共感の谷 → 希望のピーク）
# + 2点: 専門医/TV出演の権威が最初の1/3以内に登場

# ── GO判定 ──
GO条件: total_score ≥ 80 AND mCVR_score ≥ 75 AND landing_cvr_score ≥ 70
BLOCK条件: 薬機法NGが1件でもあれば固定（スコアに関わらず）

Requirements Spec
要件定義書 × エージェント別
01記事フック診断士 — 要件定義
分類要件優先度
機能 冒頭3行（または冒頭200文字）を読んで「この読者が次の行を読む確率」をスコア化する MUST
機能 悩みの解像度を評価する（「疲れやすい」は低解像度 / 「40代 膝が内側から痛む」は高解像度） MUST
機能 N1自己投影度を評価する（「私のことだ」と感じる具体性があるか） MUST
機能 感情温度を評価する（恐怖・罪悪感・希望・怒りのどれが使われているか） SHOULD
出力 hook_score（0-100）+ 改善案（MAX1文）+ 競合との比較コメント1行 MUST
制約 出力は JSON + 人間向け1文のみ。長文解説は禁止。 MUST
02記事ナラティブ診断士 — 要件定義
分類要件優先度
機能 感情アーク（共感→問題→新認識→解決→希望）の流れが正しく設計されているか判定 MUST
機能 「中だるみゾーン」を記事の何%地点かで特定する（例: 35-50%地点で感情温度が下がっている） MUST
機能 読者が「なるほど」と感じる「新認識フック」が何回仕込まれているか数える SHOULD
出力 narrative_score（0-100）+ 離脱予測ポイント + アーク評価（V字/平坦/下降） MUST
03記事信頼設計士 — 要件定義
分類要件優先度
機能 権威の種類（医師/専門家/TV出演/資格）と登場タイミング（記事の何%地点か）を記録 MUST
機能 数値証拠の質を評価（出典あり/なし、数字の具体性、社内調査 vs 第三者機関） MUST
機能 口コミの属性具体性を評価（「30代 膝が悪い 孫と遊べなかった」 vs「使ってよかった」） MUST
出力 trust_score（0-100）+ 三層スコア内訳（権威/証拠/口コミ）+ 最優先改善1点 MUST
04記事CTA診断士 — 要件定義
分類要件優先度
機能 CTA の数・配置位置（記事の何%地点）・文言を全部リストアップする MUST
機能 「CTAが必然に感じるか」を評価。直前の感情温度との連動性をチェック。 MUST
機能 アンケート型/クーポン型/限定型 の有無をチェック。使われていれば加点。 SHOULD
出力 cta_score（0-100）+ CTAリスト + 最も弱いCTAへの改善案1点 MUST
05記事オファー診断士 — 要件定義
分類要件優先度
機能 初回価格・通常価格・割引率を抽出し「価格の魅力度」をスコア化 MUST
機能 緊急性（期間限定・在庫わずか）と希少性（限定〇名・今だけ）の実装状況を評価 MUST
機能 特典・ボーナス・縛りなし設計の存在と訴求強度を評価 SHOULD
出力 offer_score（0-100）+ 価格情報サマリ + 最優先改善1点 MUST
06記事競合診断士 — 要件定義
分類要件優先度
機能 benchmark.json から同ジャンルの勝ちパターンを読み込み、実装率を%で算出する MUST
機能 今週伸びているTOP競合との差分を「このLPにないもの」として最大3点リストアップ MUST
機能 好調競合記事3本のURL・広告文・FVフックを提示する SHOULD
出力 competitive_score（0-100）+ 勝ちパターン実装率 + 競合差分TOP3 + 参考URL MUST

Test Spec
テスト指示書
フック診断士
テスト1: 高解像度フックに高スコアを出すか
入力: 「ひざ関節症専門医『膝が悪い人は絶対やって！』1日3秒の裏ワザで激変」
期待: hook_score ≥ 85（数字・専門家・行動コスト低さ）
テスト2: 低解像度フックに低スコアを出すか
入力: 「最近お体の調子はいかがですか？」
期待: hook_score ≤ 40
ナラティブ診断士
テスト1: V字アークに高スコア
入力: 共感FV → 問題深堀り → 新認識 → 解決 → 感動口コミ → CTA の構成
期待: arc_type="V字" / narrative_score ≥ 80
テスト2: 中だるみを検出できるか
入力: FV後に商品成分の説明が3,000字続くLP
期待: dropoff_point "30-60%付近" / narrative_score ≤ 55
信頼設計士
テスト1: 権威トリプルを正しく検出するか
入力: 「NHK出演・東大院卒医師監修 + 医師94%推奨 + 口コミ50件（属性あり）」
期待: trust_score ≥ 85 / 権威・証拠・口コミ全て高評価
テスト2: 権威なしを正しく低評価するか
入力: 自社比のみ / 口コミ「よかったです」のみ
期待: trust_score ≤ 50
CTA診断士
テスト1: アンケート型CTAに加点するか
入力: 「5問のアンケートに答えると98%OFFクーポン」型CTA
期待: cta_score ≥ 85 + bonus_survey=true
テスト2: CTA0本のLPを低評価
入力: 商品説明のみ、遷移ボタンなし
期待: cta_score ≤ 30
オファー診断士
テスト1: 高割引率を正しく評価するか
入力: 「通常11,810円 → 初回210円（98%OFF）定期縛りなし」
期待: offer_score ≥ 90 / urgency=true / binding=false
テスト2: 定価のみのLPを低評価
入力: 「定価5,000円（税込）」のみ
期待: offer_score ≤ 40
CMO（統合）
テスト1: GOを正しく出力するか
入力: 全エージェント基準クリア（全スコア≥75）
期待: final_verdict="GO" / must_fix=[]
テスト2: BLOCKが最優先か
入力: total_score=90 AND 薬機法NG1件
期待: final_verdict="BLOCK"（スコアに関わらず）
