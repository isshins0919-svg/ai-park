Core Philosophy
このシステムが守る4つの思想
スコアを出すだけのシステムは作らない。「何を直せばCVRが上がるか」を、CVR影響度の根拠つきで出力する。
🎯
数字が動かないFBは出さない
「全体的によくできています」「〜を検討してもよいかもしれません」は禁止。 must_fixには必ず「[mCVR+X%推定]」か「[着地CVR+X%推定]」を付ける。 推定根拠がないなら言わない。
⚡
修正工数×CVR効果 = ROI順優先
must_fixは「影響度が大きいもの」ではなく「影響度 ÷ 修正工数」が大きいものから出す。 2時間かけて+10%より、10分で+8%のほうが先に直すべきだ。
🧠
6つの視点を並列で持つ
人間のライターは記事を書きながらすべてを最適化できない。 フック・アーク・信頼・CTA・オファー・競合の6軸を同時に見るのがAIの強み。 1人のCMOが6人の専門家を率いて統合判断を出す。
📈
記事を作るAIとの未来統合
現在はFBツール。しかし将来的には「記事生成AI → CMO審査 → GO出たら公開」の全自動ループを目指す。 CMOがGOを出した記事だけが世に出る仕組みへ。

Agent Lineup
7つの専門エージェント
AGENT 00 — 統括
記事CMO君 👑

AGENT 01 — Group A
記事フック君
mCVR 直結
冒頭200文字で「私のことだ」と思わせるか。悩み解像度・N1自己投影度・感情温度・数字パワーの4軸でスコア化。
hook_score ≥ 75
AGENT 02 — Group A
記事アーク君
mCVR + 着地CVR
感情アーク（共感→問題→新認識→解決→希望）の設計を評価。中だるみゾーンを%で特定。新認識フックの数をカウント。
narrative_score ≥ 70
AGENT 03 — Group B
記事信頼君
着地CVR 直結
権威（医師・TV・専門資格）×証拠（第三者データ）×口コミ（属性具体性）の3層で信頼スコアを算出。権威の登場タイミングも評価。
trust_score ≥ 70
AGENT 04 — Group B
記事CTA君
mCVR 直結
CTAの数・配置・文言強度・直前感情との必然性を評価。アンケート/クーポン型の有無をチェックして加点。
cta_score ≥ 80
AGENT 05 — Group B
記事オファー君
着地CVR 直結
初回価格・割引率・緊急性・希少性・縛りなし設計の5点で購買衝動スコアを算出。縛りありは即マイナス評価。
offer_score ≥ 70
AGENT 06 — Group C
記事競合君
mCVR + 着地CVR
benchmark.jsonから同ジャンルTOP競合の勝ちパターンを抽出し、実装率を%算出。「この記事にないもの TOP3」を定量化。
competitive_score ≥ 65
Score Design
スコア計算式 × CMO重み付け
mCVRと着地CVRをそれぞれ独立して測定し、最後に50:50で統合する。どちらか一方が弱ければGOは出ない。
mCVR Score（遷移率に直結）
mCVR_score =
  hook_score × 0.40 # 止まるか？ 最重要
  + narrative_score × 0.35 # 読み続けるか？
  + cta_score × 0.25 # 必然的に押すか？

# ボーナス
# +2: フックに数字あり
# +2: アンケート/クーポン型CTA
# +3: V字感情アーク確認
着地CVR Score（購入率に直結）
landing_cvr_score =
  offer_score × 0.40 # 今すぐ買いたいか？
  + trust_score × 0.35 # 信頼できるか？
  + competitive_score × 0.25 # 勝ちパターン実装率

# 統合
total_score = mCVR × 0.5 + landing_cvr × 0.5

GO: total ≥ 80 AND mCVR ≥ 75 AND lcvr ≥ 70
BLOCK: 薬機法NG 1件でも → 即確定

🔧 Must Fix — ROI順（最大3点）
① [mCVR+8%推定 / 修正10分]

Deep Insights — 本質的な改善思考
v1を超えるための6つの問い
現在のv1は「診断」ができる。次に問うべきは「なぜそう診断したか」「その診断は正しかったか」「ジャンルによって何が違うのか」。
01
v2 候補
重みはジャンルで変わるはず
現在のmCVR式はhook×0.40 + arc×0.35 + cta×0.25で固定。 しかし健康×高齢者ジャンルは「信頼」が最重要、美容×20〜30代は「共感フック」が最重要のはず。 ジャンル別に重みを変えるだけで、スコアの精度が劇的に上がる。

→ benchmark.jsonにジャンル別重みテーブルを追加し、記事CMO君が自動で切り替える。
02
v2 候補
ROI順のmust_fixがない
現在のmust_fixは「CVR影響度順」。でも「修正工数」が考慮されていない。 「フック1行書き換え（10分）→ +8%」より「権威写真差し替え（2時間）→ +10%」を先に並べるのは間違い。

→ must_fixにeffort_min（推定分数）を追加し、ROI = 影響度 ÷ 工数で並び替える。 これがあるとライターが「今日何をやるか」をAIが決めてくれる状態になる。
03
v2 候補
仮説検証ループがない
CMOが「フックを変えると+8%」と言った。でも本当に+8%になったか、誰も追っていない。 これでは「当たってる予測をするエージェント」が育たない。

→ FB後に記事が修正されてURLが更新されたら自動で再評価し、スコア差分を記録。 「記事Aはフック改善で+12%、平均予測+8%より高かった」を蓄積し、 そのジャンルの次のFBに活かす。
04
v2 候補
テキストしか見ていない
全6エージェントがテキスト情報だけを解析している。でも実際のCVRはビジュアルにも依存する。

・CTAボタンはファーストビュー内に見えるか？
・ヒーロー画像の感情トーンとテキストが合っているか？
・信頼バッジは目に入りやすい位置にあるか？

→ v2でPlaywrightスクリーンショット + Vision AIによる「記事ビジュアル君」を追加。
05
v3 候補
競合ベンチマークが静止画
現在のbenchmark.jsonは手動更新。DProで1位が入れ替わったとき、既存記事には何も起こらない。

→ 週次でbenchmark.jsonを自動更新し、「前週比でrank1が変わったジャンル」の既存記事に自動再審査アラートを出す。 「先週あなたが作ったプルーストの記事、競合が変わりました。再審査します」が飛んでくる。
06
v3 候補
記事生成AIとの統合
現在は「書き終わった記事をFBする」ツール。でも理想は「書く前〜書きながら〜書き終わりまで全部CMOが見ている」状態。

→ 記事生成AI（記事LP Park）が構成案を出した段階で、記事CMO君が構造レビューを実施。 「この構成ではmCVRが取れない、先にフック案を3パターン用意してから書き始めて」が先回りで来る世界へ。 FBは後処理から前処理へ。
Future Agents
v2/v3で追加される君たち
v2 予定
記事ビジュアル君
スクリーンショット + Vision AI。CTAボタンがFV内に入っているか、ヒーロー画像の感情トーン、信頼バッジの視認性を評価。
v2 予定
記事スピード君
読了予測時間 × 感情ペイオフ曲線を計算。「3,000字で1つも感情山が来ない」セクションを検出して削除推奨。
v2 予定
記事薬機法君
現在CMO君が簡易チェックしている薬機法を専門エージェント化。「治る」「必ず」等の危険表現を全文スキャン。BLOCK判定の精度を上げる。
v3 候補
記事学習君
FB後の修正URLを自動再評価し、スコア差分を記録。「どのエージェントの改善提案が最も実際のCVR向上に貢献したか」を学習し、重みを自動チューニング。
v3 候補
記事競合アラート君
週次でbenchmark.jsonを自動更新。同ジャンルのrank1が入れ替わったら既存記事に自動再審査依頼を送信。競合に追い抜かれたら即アラート。
v3 候補
記事コーチ君
記事LP Parkが構成案を出した段階で記事CMO君にパス。「この構成ではmCVRが取れない」を書き始める前に指摘。FBを後処理から前処理へ。
