## Phase 2: embedding-2 マルチモーダルベクトル化

### 手法
- モデル: `gemini-embedding-2-preview` (3072次元)
- 3ベクトル/バナー: image_vec / text_vec / multimodal_vec
- 画像+テキストを同一Contentに含めた真のマルチモーダルembedding

### 重要発見: 閾値キャリブレーション問題
```
広告バナー間のコサイン類似度: 0.50〜0.80に集中
従来の多様性フィルタ(<=0.70)が機能しない
→ 広告ドメイン専用の閾値再設定が必要
推奨: 同カテゴリバナー同士は0.65〜0.75が「類似」の実態
```


## Phase 5 POC 総括

### このPOCで証明したこと

1. **embedding-2マルチモーダルは分析ツールとして有効**  
   画像+テキスト統合embeddingでクラスタ分離精度が大幅改善（8:1:1 → 3:6:1）。
   ただし広告ドメイン専用の類似度閾値キャリブレーションが必要。

2. **日本語バナー生成はgpt-image-2一択（2026-04時点）**  
   Gemini Imagen 4 は日本語テキスト生成が実用不可レベル。
   gpt-image-2 は日本語コピー/キャラ/ロゴを正確に統合した広告バナーを生成可能。

3. **N1フックの修正が最重要**  
   「失敗したら悲しい」→「自分でらぶいーずを編める発見のワクワク」へのフレーミング転換が
   バナーコピー全体の設計を変えた。発見→欲しい の順が正しい。

4. **DPro ジャンル体系の限界**  
   「推し活/ぬいぐるみ/手芸」系バナーはDPro在庫が極少。
   近傍ジャンルからの転用 + keyword検索非機能の2重ハンデ。
   次回: 類似案件の手動収集フローを整備する。
