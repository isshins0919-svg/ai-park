## 2. 最大の武器 = データ駆動（§1 末尾のハイライト）

> 「普通のAIに『バナー作って』と言うと、なんとなく良さそうなコピーで出力します。AIパクくんは違います。競合広告を数百件分析し、全データをEmbedding（ベクトル化）してから、データに裏付けられたコピーでCRを作ります。これが **大当たりCRと凡作の分かれ目** です。」

マニュアルは **「Embeddingがあるかないか」を最大の訴求軸**としている。

### Embeddingなし vs あり（§5 の対比表）

| 項目 | Embeddingなし | Embeddingあり |
|---|---|---|
| コピーの根拠 | AIの一般知識（なんとなく） | 競合の勝ちCR分析結果 |
| N1の精度 | 推測ベース | データから導出 |
| コンセプト | ありきたり | 市場の穴を突く |
| 結果 | 凡作 | 大当たりCR |

## 3. チーム紹介（§2, 9人掲載）

マニュアルに顔写真付きで紹介されている「AIパクくんたち」:

### 経営陣

1. **CEO AIパクくん** — あなたが話しかける相手
   - デフォルト画面に常駐
   - エージェントの設計・改善・仕組みづくりが得意
   - マーケティングの仕事はCMOに振り分ける

2. **CMO AIパクくん** — マーケティングの総指揮官
   - リサーチ → コンセプト → 制作 → 品質チェック
   - 必要なAIパクくんを適材適所で動かす

### 専門チーム

3. **RAG AIパクくん** — データの番人（最重要）
   - 「AIパクくんの心臓部」
   - 広告データ・LP・競合情報を全てEmbedding 2（3072次元ベクトル）に変換
   - RAGなしにCRは作れない

4. **コンセプト AIパクくん** — 戦略の設計者
   - 「誰に・何を・どう伝えるか」をデータから導出
   - N1ごとに独立したコンセプトを設計
   - 各コンセプトのオリジナリティを最大化

5. **N1 AIパクくん** — ターゲット顧客の専門家
   - ペルソナの痛み・欲求・日常シーン・購買心理を分析
   - 「この人がこの瞬間にこの広告を見たら動く」レベルまで解像度

6. **ショート AIパクくん** — 動画広告の職人
   - 台本設計、シーン構成、テロップ設計、Remotionプロジェクト生成
   - TikTok・Reels・Shorts対応

7. **記事LP Writer AIパクくん** — コピーライター
   - 記事LPの執筆を担当
   - データに基づいたコピーライティングで読者心理を段階的に動かす

8. **記事LP AB AIパクくん** — A/Bテストの科学者
   - 既存A記事LPからB案を設計
   - 「何を変えれば成果が変わるか」を科学的にアプローチ

9. **入稿 AIパクくん** — デプロイの実行者
   - 完成したCRをMeta広告等に入稿する準備

10. **運用 AIパクくん** — 広告運用のモニター
    - CPA/CTR異常検知 → 改善アクション提案

※ これは **顔アイコン付きのユーザー向け公開メンバー**。実際のパッケージには 38体のAgent定義ファイルがあるので、裏方Agent（Director / dprokun / rag-2 / ops 等）は非表示。

## 4. セットアップ（§3）

### Mac

```
Step 1: Claude Code インストール（claude.com/download, Pro $20/月 or Max $100/月）
Step 2: AIパクくんzipを展開 → ターミナルで cd ~/Desktop/AIパクくん1.2.4 && claude
Step 3: Claude Code内で /setup を実行
```

### Windows

```
Step 1: Git for Windows インストール（必須）
Step 2-4: Mac と同じ
```

### 重要注意: **APIキーは絶対にチャット欄に貼らない**
- チャット履歴に残り、セキュリティリスク
- 必ず `.env` ファイルに記入
- `/setup` がサポート

## 5. 使い方（§4）

### 推奨: 自然言語で依頼

CEOに話しかけるだけ。CMOが自動でチームを指揮する。

**プロンプト例1: バナー広告を一括生成**
```
https://example.com/lp この商品のバナー広告を作ってください。
商品リサーチをしっかり行い、RAGを構築して、
N1を4パターン選定して、それぞれにコンセプトを作ってください。
そのあとコンセプトごとに8ずつバナーを、合計32本作ってください。
できたらインターンパクくんにCRをまとめてもらってください。
```

**プロンプト例2: 競合分析**
```
この商品の競合を調べて、勝っている広告の共通点を分析してください。
https://example.com/product
どんなコピーが成果を出しているか、市場全体の傾向を教えてください。
```

**プロンプト例3: 改善提案**
```
先月のバナー広告の結果データを渡します。
勝ちCRの共通点と負けCRの問題点をデータから分析して、
新しいコンセプトを3つ提案してください。
```

**プロンプト例4: 記事LP**
```
この商品の記事LPを作ってください。
https://example.com/lp
ターゲットは30代女性、肌悩みを持つ方。
```

**プロンプト例5: 動画広告**
```
この商品のショート動画広告を作ってください。
https://example.com/lp
TikTok向け、15秒、縦型。
```

### スキル直接起動（工程ごと確認したい場合）

```
/research-park https://example.com/lp  ← 商品・競合リサーチ
/concept-park example-product           ← N1選定 + コンセプト設計
/banner-park example-product            ← バナー生成
/shortad-park example-product           ← 動画広告生成
/article-lp-park example-product        ← 記事LP生成
```

**警告: この順番は省略できません**
> `/research-park → /concept-park → /banner-park` の順番が品質を決めます。

## 12. VOYAGE号にとってのマニュアルからの学び

### 学び1: ユーザー体験の簡素化
- VOYAGE のCLAUDE.md・各スキルは技術情報が豊富
- パクのマニュアルは **「本当に必要な情報だけ」に削ぎ落とし**
- VOYAGE もエンドユーザー向けガイドと開発者向けドキュメントを分離する価値あり

### 学び2: 「AIに顔を与える」
- パクは9人の Agent に顔イラストを与えてキャラ化
- VOYAGE は絵文字（⚓🔥🎣⚡🪝🧭）でクルーを識別しているが、ビジュアル化は未実装
- クライアント納品時の提案書に「VOYAGE クルー紹介」を入れる価値あり

### 学び3: 「最大の武器」を1つに絞る訴求
- パクは「データ駆動（Embedding 2）」を最大の武器と明確化
- VOYAGE は特徴が多すぎて訴求軸がぼやけがち
- VOYAGE の場合: **「戦略翻訳 × 複利ループ」**が最大の武器として明確化できる

### 学び4: 「困った時は」フローの一本化
- `/people-park` でエラー自動収集 → 確認画面 → 送信
- VOYAGE もトラブルシューティング導線を1スキルに集約する価値あり
- 既存の `/handoff` や `/nice-dive` と組み合わせた「/help-me」的スキルを検討

### 学び5: 順番の守護
- マニュアル明記: 「**この順番は省略できません** `/research-park → /concept-park → /banner-park`」
- ユーザーは順番を守らない傾向がある
- VOYAGE のスキルも前提チェック（strategy.json有無 / RAG鮮度）を強化する価値あり
