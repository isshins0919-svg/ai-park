# 動画制作チーム 技術環境レポート

> 作成日: 2026-04-04 | 作成: 一進AI（動画カントククルー主導）

---

## 1. エグゼクティブサマリー

当チームの動画制作は、**AI駆動の統合パイプライン**により、「編集依頼（台本CSV）」から「完成動画（mp4）」までを**ほぼ全自動**で完結させています。

- **10種のAIエージェント**が台本の品質を多角的に評価
- **5つの外部AI**（Claude / Gemini / Fish Audio / Grok / Nano Banana）を組み合わせた制作パイプライン
- **品質ゲート制度**により、基準未達の動画は自動ブロック
- 人間の作業は**台本ドラフト作成とテーマ決定のみ**（全体の約10%）

## 2. 制作フロー全体像

```
【人間】台本CSV作成（テキスト・素材指定・注釈）
    |
    v
【AI】カントククルー（オーケストレーター）が受け取り
    |
    v  ── 並列起動 ──
    |
    |  [Group A: 3クルー並列]
    |   ├─ フッククルー ─────── 冒頭0-4秒の「止まる力」評価
    |   ├─ アーククルー ─────── 60秒全体の感情設計評価
    |   └─ リテンションクルー ── 視聴維持率の予測
    |
    |  [Group B: 4クルー並列]（Group A完了後）
    |   ├─ テンポクルー ─────── スロット配置の最適化
    |   ├─ スタイルクルー ────── テロップの強調配分
    |   ├─ CTAクルー ─────────── 行動喚起の設計評価
    |   ├─ マッチクルー ─────── 映像×テキストの整合性
    |   └─ ブリッジクルー ────── 動画→LP遷移の連続性
    |
    v
【AI】ジャッジクルー ── 総合スコア算出 → GO / REVISE / BLOCK 判定
    |
    | GOの場合
    v
【AI】edit_ai_v2.py ── 動画自動生成パイプライン
    |
    ├─ Gemini Vision: 素材クリップの自動分析
    ├─ Fish Audio: ナレーション音声の自動生成
    ├─ OpenCV/ffmpeg: 縦型クロップ・向き補正
    ├─ テロップ自動生成（3スタイル）
    ├─ BGM自動選定・音量最適化
    └─ moviepy: 全素材を結合 → mp4出力
    |
    v
【完成】1080x1920 縦型動画（60秒 or 30秒）
```

## 4. AIが持つ学習・知識体系

### 4.1 ナレッジベース（蓄積データ）

| ナレッジ | 内容 | 活用場面 |
|---------|------|---------|
| **フックDB** | 実証済みフック8パターン（H1-H8） | 冒頭テキスト設計 |
| **CTA DB** | 実証済みCTA8パターン（C1-C8） | 行動喚起テキスト設計 |
| **HIT広告DNAテンプレート** | 勝ちパターンA-G（構成・接続詞・トーン） | 台本構成の自動設計 |
| **BGM選定マップ** | 商材カテゴリ別の最適BPM・ジャンル・音量設定 | BGM自動選定 |
| **シーンロールタグ** | 15種のシーン役割定義（HOOK/PAIN/PIVOT等） | テンポ最適化 |
| **モーションパターンDB** | 画像生成プロンプト構造・Ken Burns効果設定 | 映像生成 |
| **マーケティング哲学** | N1ペルソナ・コンセプト一貫性の判断基準 | 全クリエイティブの品質判断 |

### 4.2 DPro連携（外部広告データベース）

- 同ジャンルのHIT広告TOP5-10を自動取得
- 再生数・いいね数・推定広告費から「勝ちパターン」を逆算
- シーン構成・接続詞・フック型・CTA型をDNA分析して抽象化
- **データドリブンで「何が効くか」を学習し続ける仕組み**

### 4.3 品質判定の学習基準（ジャッジシステム）

**総合スコア計算式:**
```
total_score =
    フック × 25% + 感情アーク × 20% + CTA × 20% + ブリッジ × 15%
  + テンポ × 10% + スタイル × 5% + マッチ × 5%
```

**GO条件（全て満たす必要あり）:**
- 総合スコア 80点以上
- ブリッジスコア 75点以上（LP連携がブロッカー）
- QUALIFY判定 = OK（ターゲット絞り込み成功）
- 薬機法チェック = PASS

**判定フロー:**
| 判定 | 条件 | 処理 |
|-----|------|------|
| **GO** | 全条件クリア | 自動で動画生成開始 |
| **REVISE** | 一部基準未達 | 最大3点の修正指示を自動生成 |
| **BLOCK** | 薬機法NG | 全テキスト再確認 |
| **EXIT** | 3回修正してもGO出ず | 台本自体の再設計を要請 |

## 6. 動画生成パイプライン（edit_ai_v2.py）技術詳細

### 6.1 基本スペック

- **コード量**: 1,789行（Python）
- **出力**: 1080x1920（縦型9:16）/ H.264 / 30fps
- **標準尺**: 60秒（30スロット x 2秒）or 30秒（15シーン）

### 6.2 処理パイプライン

```
Step 1: CSVパース
  └─ 台本CSVを読み込み、Sceneオブジェクトのリストに変換

Step 2: Gemini Vision 分析（並列処理）
  ├─ 各クリップから複数フレームを抽出
  ├─ Geminiに送信 → 被写体・最適クロップ位置・ベスト開始秒を取得
  └─ JSON堅牢パーサで安定したレスポンス処理

Step 3: Fish Audio TTS生成（シーン単位同期方式 ← 核心技術）
  ├─ 全文一括ではなく「シーン毎」にテキストと読む秒数を同期
  ├─ 目標60秒に収まるよう読速を事前計算（0.5x〜2.0x）
  ├─ 各シーン音声生成後、無音トリムで時間調整
  └─ キャッシュ機構で再生成を回避

Step 4: クリップ処理（並列処理）
  ├─ ffmpegで向き正規化（回転メタデータ焼き込み）
  ├─ OpenCVで顔検出 → クロップ中心位置決定
  ├─ 縦型9:16にクロップ（横長素材はレターボックス対応）
  └─ テロップ画像生成・描画（3スタイル: emphasis/normal/small）

Step 5: moviepy結合
  ├─ 映像 + テロップ + ナレーション + BGM を時間同期で合成
  └─ mp4出力（H.264, 1080x1920, 30fps）
```

### 6.3 テロップシステム

| スタイル | 用途 | フォントサイズ | 色 |
|---------|------|-------------|-----|
| **emphasis** | フック・数字・CTA・商品名 | 84pt | 黄色背景+黒文字 |
| **normal** | 説明・共感・情報 | 76pt | 白文字+縁取り |
| **small** | 注釈・補足・法的表記 | 小 | 白文字+縁取り |

### 6.4 BGM自動選定・音量制御

**商材別に最適なBGMを自動選定:**

| 商材カテゴリ | BPM範囲 | ジャンル |
|-----------|---------|---------|
| 健康食品・サプリ | 95-115 | Acoustic Pop |
| 美容・スキンケア | 80-105 | Ambient Pop / Lo-Fi |
| ダイエット | 120-140 | EDM / Upbeat Pop |
| BtoB SaaS | 85-100 | Corporate / Minimal |
| EC・ファッション | 110-130 | Hip-hop / Trap Pop |

**フェーズ別音量自動調整:**

| フェーズ | BGM | ナレーション | 設計意図 |
|---------|-----|------------|---------|
| STOP（0-3秒） | -14dB | -3dB | 冒頭は音圧で引く |
| QUALIFY（3-8秒） | -20dB | -6dB | VO優先・BGMは背景 |
| BUILD（8-25秒） | -18dB | -3dB | 説明パートは聞きやすく |
| DRIVE（25-60秒） | -12dB | -3dB | 終盤でエネルギー最大化 |

## 7. 品質管理体制

### 7.1 多層評価システム（10クルー体制）

```
                     ┌──────────────┐
                     │  カントク    │ ← 全体統括
                     │（orchestrator）│
                     └──────┬───────┘
                            │
          ┌─────────────────┼─────────────────┐
          │                 │                 │
    ┌─────┴─────┐    ┌─────┴─────┐    ┌─────┴─────┐
    │ 視聴継続  │    │  遷移率   │    │  LP CVR   │
    │  評価群   │    │  評価群   │    │  評価群   │
    └─────┬─────┘    └─────┬─────┘    └─────┬─────┘
          │                 │                 │
    フッククルー        CTAクルー        ブリッジクルー
    アーククルー        QUALIFY判定      マッチクルー
    テンポクルー        DRIVE判定
    スタイルクルー
    リテンションクルー
          │                 │                 │
          └─────────────────┼─────────────────┘
                            │
                     ┌──────┴───────┐
                     │  ジャッジ    │ ← 最終判定
                     │ GO/REVISE   │
                     │ BLOCK/EXIT  │
                     └──────────────┘
```

### 7.2 薬機法チェック

- 全テキストに対して薬機法準拠チェックを自動実施
- NG表現検出時は**BLOCK判定**（手動修正まで進行停止）
- 健康食品・美容・サプリ案件で特に厳格に適用

### 7.3 ブリッジスコア（動画→LP連携）

**最も重視する指標。** 動画単体の品質より「LPへ正しく送り込めるか」を優先。

| チェック項目 | 重み | 内容 |
|------------|------|------|
| テーマ連続性 | 30% | 動画の出口とLPの入口が繋がっているか |
| トーン連続性 | 25% | 感情温度（緊急感・共感）が一致しているか |
| ペルソナ一致 | 25% | 動画のターゲットとLP読者が同一か |
| ギャップ引き継ぎ | 20% | 動画で残した「未解決の問い」をLPが回収しているか |

## 12. まとめ

当チームの動画制作システムは、**「AIが品質を担保し、人間は創造的判断に集中する」** という設計思想のもとに構築されています。

**現状:**
- 台本から完成動画まで、**人間の作業は全体の約10%**
- 10種のAIクルーによる**多角的品質評価**で、主観に頼らない品質管理
- **ブリッジスコア**による動画→LP一貫性の担保

**今後:**
- 実績データのフィードバックループ構築により、**AIが自ら精度を高め続ける仕組み**へ
- 素材生成AIの進化により、**素材ゼロからの完全自動動画制作**が現実的に
- 最終的には**企画→制作→配信→分析→改善**の全サイクルをAI主導で回す未来
