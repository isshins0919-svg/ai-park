COMMANDS VS AGENTS

2種類のエージェント — commands/ と agents/

Island 04で学んだSkill（commands/）は、ユーザーが /skill名 で起動するもの。
もう1つ、AIが自動で起動するサブエージェント（agents/）があります。

commands/（Skill）agents/（サブエージェント）
起動方法ユーザーが /skill で手動AIが Agent tool で自動
コンテキスト親と共有（Tier 2）独立空間（Tier 3）
向いてる用途対話型・オーケストレーターレビュー・評価・並列タスク
人間で例えると目の前で指示を出す部下別室で作業する専門家
判断基準

ユーザーとの対話が必要 → commands/
独立して完結する専門タスク → agents/

🎬 映画制作の例

commands/ = 監督（俳優と直接やりとりする）
agents/ = 各部門の専門家（照明、音声、衣装は別室で準備）

監督が全部1人でやったら破綻する。専門家に任せるから高品質。

⚓ VOYAGE号の実例

/kiji-cko（commands/）= 記事LP制作の指揮官。ユーザーと対話しながら進行

→ kiji-hook-crew（agents/）= フック評価を独立CWで実行
→ kiji-trust-crew（agents/）= 信頼構造を独立CWで評価
→ kiji-arc-crew（agents/）= 感情アークを独立CWで分析

DESIGN PRINCIPLES

エージェント設計の7つの原則

📐 7つのルール

これを守れば、エージェントは安定して動く

1

目的・ゴール・行動指針・制約条件を必ず持たせる

人間の組織と同じ。役割が曖昧だと暴走する。

2

1エージェント1タスク

複数タスクを持たせるとコンテキストが溢れて品質が落ちる。

3

まず1体動かしてから組織を拡大

最初から大組織を作らない。1体が安定してから次を作る。

4

新エージェントは上司エージェントにレビューさせる

人間がレビューするより、関連エージェントにレビューさせた方が品質が高い。

5

自走ルールを渡す

「3回自分で試してから聞く」等のルールで、不要な確認を減らす。

6

制約は後入れ

最初は自由に作らせる → 後から法規制等のチェックを入れる。最初から制約をかけると守りに入る。

7

データを与える（エンベディング）

200本のデータがあれば十分な精度。データの質（勝ち/負けのラベル）が量より重要。

SPLITTING SIGNALS

いつエージェントを分割するか — 3つのNG信号

1体に全部やらせると品質が崩壊します。以下の信号が出たら分割のサイン。

🚨 3つのNG信号

これが出たら、1つに詰め込みすぎ

!

ループ3回以上NG

同じミスを修正しても3回繰り返す → タスクが複雑すぎるか、CWが圧迫されている

!

指示忘れが発生

最初に伝えた条件が途中から無視される → CWの初期部分に注意が行き届いていない

!

ハルシネーションが発生

存在しない情報を「ある」と言い始める → 脳がパンクしている明確なサイン

🏥 病院の例

1人の医師が外来・手術・入院管理・書類作成を全部やったら？
→ ミスが起きる。だから分業する。

外来担当 / 手術担当 / 事務 = エージェント分割

⚓ VOYAGE号の分割例

記事LP制作:
/kiji-cko が全体を指揮（commands/）
→ hook-crew / trust-crew / arc-crew / offer-crew / cta-crew が各領域を独立評価（agents/）

各クルーはTier 3で動くから親のCWを消費しない。

SUMMARY

Island 05 まとめ

🏗️
3つの持ち帰り
① commands/（対話型）と agents/（独立型）の使い分けを覚える
② 7つの設計原則: 1タスク1体、自走ルール、制約は後入れ
③ NG信号3つ（ループ3回NG・指示忘れ・ハルシネーション）が出たら分割する

🏁 チェックポイント — この島で覚えること

  • commands/ = 対話型、agents/ = 独立型。コンテキストの管理方法が違う
  • 1エージェント1タスクが基本。欲張らない
  • 3つのNG信号（ループ・忘却・ハルシネーション）で分割を判断する
  • まず1体を安定させてから組織を拡大する

📖 この島の航海用語

agents/

サブエージェントの定義ファイルを置くフォルダ。独立コンテキストで動く。

オーケストレーター

全体を指揮するメインのSkill（commands/に置く）。ユーザーと対話する。

サブエージェント

専門タスクを独立で実行するAI。agents/に置く。親のCWを消費しない。

Tier 2

commands/の情報コスト。親のコンテキストに読み込まれる。重いスキルは注意。

Tier 3

agents/の情報コスト。独立空間で動くため、親のCWに影響なし。

NG信号

エージェント分割が必要なサイン。ループ3回NG・指示忘れ・ハルシネーションの3つ。

🏗️ エージェント組織図を設計しよう

自分の業務チームをAIエージェントの組織図として設計するハンズオン。
starter-kit の agent-design-principles.md を参考にします。

ゴール

自分の業務に必要なエージェント組織図（オーケストレーター + サブエージェント）を設計する

  1. Island 01で作った「7パーツの一覧表」を振り返る
  2. 自分の業務タスクを「対話が必要」vs「独立で完結」に分類する
  3. Claude Codeで以下を実行:
    starter-kit/agent-design-principles.md を読んで、
    以下の業務のエージェント組織図を設計してください。

    # 業務内容
    - 部署: [あなたの部署]
    - 主要タスク: [3-5個列挙]
    - 使うツール: [Slack, Drive, etc.]

    # 出力形式
    - オーケストレーター（commands/）: 何を担当？
    - サブエージェント（agents/）: 各1タスクで設計
    - 各エージェントの役割・ゴール・制約を明記
  4. 「7つの設計原則」に照らしてレビュー。特に「1体1タスク」を守れているか確認
所要時間: 約25分
