AIエージェントの正体

7つのパーツを知れば、全体像が見える

本編 15分 + ミッション 15分
BEFORE WE START

そもそも「AIエージェント」って何？

「ChatGPTに質問すること」と「AIエージェントを使うこと」は全く違います。

ChatGPT（チャットAI）

人間が質問 → AIが回答 → 人間が次を質問

AI = 辞書。聞いたことに答えるだけ。
ファイルを読めない。メールを送れない。データベースにアクセスできない。

AIエージェント

人間が目的を伝える → AIが自分で考え → 自分で実行 → 結果を報告

AI = 新入社員。指示を受けて自分で動く。
ファイルを読み書き。Slackに投稿。カレンダーを確認。全部自分でやる。

🏢 オフィスで例えると

ChatGPT = 百科事典に詳しい人がデスクに座ってるだけ。「これ何？」に答えてくれるけど、電話もメールもできない。

エージェント = PCもSlackもカレンダーも使える新入社員。「この案件の競合調べて、レポートにまとめて、Slackに投げといて」ができる。

⚓ VOYAGE号での実例

「大当たりクリエイティブを生み出し続ける」というゴールを持ち、自分でリサーチして、コンセプトを作り、バナーを生成する。人間は最終チェックだけ。

どんなに複雑なAIエージェントも、分解すると7つのパーツに分かれます。

🧠 1. LLM — 脳みそ

考える力そのもの。でも「脳だけ」では何もできない

LLM（Large Language Model）はAIの「脳」。テキストを理解し、推論し、文章を生成する力の源泉。Claude、GPT、Geminiなどがこれにあたります。

ただし、脳だけでは現実世界に手出しできない。ファイルを読めない、メールを送れない、データベースにアクセスできない。脳には「手足」が必要です。

なぜ大事？

LLMの性能がエージェント全体の「思考力の上限」を決めます。脳の能力に合った道具を選ぶのが設計の第一歩。

🍳 料理の例

世界一のシェフでも、包丁もコンロもない部屋では料理できない。逆に最高のキッチン設備があっても、レシピを知らなければ使えない。
脳（シェフの腕前）× 道具（キッチン設備）= 料理の質

⚓ 実例

Claude Opus 4.6（100万トークンCW）を使用。現時点で最高峰のLLM。「脳」が強力なので、複雑な分析・判断・文章生成が1セッションでできる。

🔧 2. Tools（MCP）— 手足

AIが「実際にやる」ための道具。接続先が増えるほど能力が広がる

MCP（Model Context Protocol）は、AIが外部サービスを操作するための統一規格。Slack送信、Google Calendar確認、ファイル操作、ブラウザ操作...AIに「手足」を与える仕組みです。

なぜ大事？

ChatGPTに「メール送っといて」と言っても送れません。Claude Codeに「Slackに投稿して」と言ったら本当に投稿できます。この差がTools（MCP）。AIが「考えるだけ」から「実行できる」に進化するかどうかの分水嶺です。

📱 スマホの例

アプリが入っていないスマホは、電話しかできない。LINEを入れれば連絡できる。カメラアプリを入れれば写真が撮れる。MCPは「AIにアプリをインストールする」ようなもの。

🔌 USBの例

USB以前は、プリンターもマウスもそれぞれ違う端子だった。USBで統一されて「何でも刺せば使える」になった。MCPも同じ。全部同じ規格でAIに繋がる。

MCP接続先何ができるか仕事で例えると
Slackメッセージ送信・チャンネル読み取り社内チャットスタッフ
Google Calendar予定確認・作成・空き時間検索秘書
Canvaデザイン生成・編集・エクスポートデザイナー
Chromeブラウザ操作・スクリーンショットWeb調査スタッフ

※ MCPの詳しい仕組みはIsland 03で深掘りします。

⚡ 3. Trigger — きっかけ

「いつ動くか」を決める。手動 → 自動化の進化ポイント

Triggerは「AIがいつ動き出すか」のきっかけ。毎回チャットに打ち込む「手動」から、時間指定・イベント検知の「自動」まで段階があります。

なぜ大事？

毎朝「今日の予定教えて」と打つのは面倒。「毎朝8時に自動で予定を教えてくれる」なら便利。人間が起動しなくても動くのが、ツール→エージェントへの進化です。

⏰ 目覚ましの例

毎朝自分で時計を見て起きる = 手動
アラームが鳴って起きる = スケジュールトリガー
スマートアラームが浅い眠りを検知して起こす = 条件トリガー

⚓ 実例

現在は手動が中心（「/morning」と打つ）。DProニュースクルーは毎朝8時に広告トレンドを自動取得してSlackに投稿する、スケジュールトリガー型。

🧭 4. Instructions — マニュアル

「言わなかったら間違う方向に進むか？」で書くかどうかを判断する

InstructionsはAIの行動を方向づける「マニュアル」。CLAUDE.mdが「必ず読むファイル」として、全セッションで適用されます。

なぜ大事？

新人に「いい感じにやっといて」と丸投げしたら自己流で事故る。でもマニュアルが100ページあったら誰も読まない。ちょうどいい量を、ちょうどいい具体度で書く。これがInstructions設計の本質。

🏗️ 設計図の例

「いい感じの家建てて」と言われたら？窓の数も間取りもバラバラな家ができる。設計図があれば、職人は迷わず正確に作れる。設計図 = Instructions。

⚓ 実例

CLAUDE.md = 全体のマニュアル
identity.md = AIの人格設定
rules/ = セキュリティ・個人情報保護の禁止事項

「言わなかったら事故る」内容だけ厳選して書かれている。

📋 5. Skills — 業務手順書

「毎回同じ説明をするなら、手順書にしよう」がSkillの発想

Skillは「再利用可能な業務手順」。毎回チャットで長い指示を書く代わりに、一度まとめておけば /skill名 で呼び出せます。

なぜ大事？

マクドナルドが世界中で同じ味を出せるのは、全工程がマニュアル化（=Skill化）されているから。属人化していないから、誰でも同品質で出せる。AIのSkillも同じ原理。

📝 チェックリストの例

パイロットが離陸前にチェックリストを確認する理由: 「毎回全部覚えているはず」は危険。手順を外出しすることで、ミスを構造的に防ぐ。

⚓ 実例

/morning — 毎朝の起動ルーチン
/banner-park — バナー制作パイプライン
/handoff — セッション引き継ぎ
/nice-dive — 学びのログ記録

💾 6. Memory — 長期記憶

「前も同じ説明したよね？」をなくす仕組み

Memoryは「セッションをまたいで覚えておく情報」。ユーザーの好み、過去の判断基準、プロジェクトの背景情報など。セッションが変わっても「いつもの」で通じる状態を作る。

なぜ大事？

行きつけのカフェで毎回「カフェラテのミルク多めで」と説明するのは面倒。常連になれば「いつもの」で通じる。AIのMemoryも「いつもの」を可能にする仕組み。

🏥 カルテの例

病院に行くたびに「アレルギーありますか？」と聞かれたら面倒。カルテがあるから、前回の情報を引き継げる。Memory = AIのカルテ。

⚓ 実例

knowledge/ = ナレッジDB（マーケ哲学、フックDB等）
clients/ = クライアント情報
memory/ = ユーザーの好み・判断記録

🪟 7. Context Window — 今の頭の中

AIの「今、頭の中に入っている量」。これが全ての品質を決める

Context Window（CW）は「AIが一度に処理できる情報量の上限」。人間の短期記憶（ワーキングメモリ）に相当します。

なぜ大事？

会議で10個の議題を一気に話されたら、最後の方は覚えていない。AIも同じ。情報を詰め込みすぎると「嘘をつく」「指示を忘れる」「精度が落ちる」が起きる。これが「ハルシネーション」の原因の一つ。

🧳 スーツケースの例

30Lのスーツケースに全部詰め込もうとすると、大事なものが奥に埋もれて取り出せない。CW = スーツケースのサイズ。詰め方の設計が命。

⚓ 実例

CW 100万トークン（業界最大級）。でも無限ではない。だからエージェントを分割して、それぞれが独立したCWを持つ設計にする（Island 06で詳しく）。

SUMMARY

Island 01 まとめ

7パーツの覚え方
脳（LLM）× 手足（Tools）× きっかけ（Trigger）= 動くAI
マニュアル（Instructions）× 手順書（Skills）= 正しく動くAI
記憶（Memory）× 頭の容量（CW）= 賢く動くAI

1LLM考える力脳みそ
2Tools / MCP実行する力手足 + 道具
3Triggerいつ動くか出勤時間 / アラーム

7Context Window今の頭の容量ワーキングメモリ

🏁 チェックポイント — この島で覚えること

  • AIエージェントは7つのパーツで構成されている
  • 「チャットAI」と「エージェント」の違いは、自分で実行できるかどうか
  • Context Window（CW）の管理がエージェントの品質を決める最重要要素

🗺️ 乗組員名簿を作ろう

自分の仕事をAIエージェントの7パーツに当てはめて整理するハンズオン。
「自分の仕事にAIを導入するなら、何が必要か？」を具体的にする最初のステップです。

ゴール

Claude Codeを使って、自分の業務に必要なAIエージェントの7パーツを書き出す

  1. Claude Codeを開く（ターミナルで claude と入力）
  2. 以下のプロンプトをそのまま貼り付ける:
    私の仕事に必要なAIエージェントを設計してください。

    # 私の仕事
    - 職種: [あなたの職種を書く]
    - よくやるタスク: [3つ挙げる]
    - 使っているツール: [Slack, Googleなど]

    # 7パーツに当てはめて
    1. LLM: どのモデルが最適？
    2. Tools: 何を繋ぐ必要がある？
    3. Trigger: いつ動かす？
    4. Instructions: 何を書く？
    5. Skills: どんな手順を定型化する？
    6. Memory: 何を覚えておく？
    7. Context Window: 何を優先的に入れる？
  3. AIの回答を読んで、自分の仕事に合っているか確認する
  4. 「これを表形式でまとめて」と追加で指示して、見やすく整理する
所要時間: 約15分
成果物

自分の業務に必要な7パーツの一覧表。これがIsland 10の最終ミッション「自分の船を造る」の設計書になります。
