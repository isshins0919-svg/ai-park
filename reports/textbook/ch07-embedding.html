THE PROBLEM

AIは「言葉」をどう理解しているのか？

🔍 ベクトル化なし vs あり

同じAIが、理解の深さだけでこれだけ変わる

ベクトル化なし

AIは言葉を「文字列」として処理する。

「膝がガクガクする」と「膝が不安定」→ 別の意味として扱う

結果: キーワードが一致しないと見つけられない。

ベクトル化あり

AIは言葉を「意味の座標」として処理する。

「膝がガクガクする」と「膝が不安定」→ ほぼ同義とわかる

結果: 言い回しが違っても「意味が同じ」を見つけられる。

WHAT IS A VECTOR

ベクトル = 意味を数値に変換したもの

「ベクトル化」とは、テキストを数値の配列（ベクトル）に変換すること。変換すると、テキスト同士の「意味の近さ」を数値で計算できるようになります。

📍 地図の例

東京の位置 = (北緯35.6, 東経139.7)
大阪の位置 = (北緯34.6, 東経135.5)
ニューヨーク = (北緯40.7, 西経74.0)

座標があれば「東京と大阪は近い」が計算でわかる。
ベクトル化は言葉に座標を与えること。

🎨 色の例

「赤」= (255, 0, 0)
「オレンジ」= (255, 165, 0)
「青」= (0, 0, 255)

数値で表すと「赤とオレンジは近い」が計算でわかる。
言葉のベクトルは3,072次元。微妙な意味の違いまで捉える。

ベクトルの距離 = 意味の近さ

「膝がガクガクする」←→「膝が不安定」 = 距離 0.15（ほぼ同義）
「膝がガクガクする」←→「足が痛い」   = 距離 0.45（関連あり）
「膝がガクガクする」←→「天気が良い」  = 距離 0.92（無関係）

→ この「距離」を使って「似たものを探す」のがベクトル検索
KEYWORD VS SEMANTIC

従来の検索 vs ベクトル検索

📚 図書館の例

従来（キーワード一致）
「犬の飼い方」で検索 → 同タイトルの本だけヒット

ベクトル検索（意味検索）
「犬の飼い方」で検索 → 「はじめてのワンちゃん」「ペットとの暮らし」もヒット

🗂️ 社内ファイルの例

従来: 「売上報告」で検索 → ファイル名一致のみ

ベクトル: 「売上報告」で検索 → 「Q3 Revenue Review」「第3四半期実績」もヒット
言語の壁すら超える。

IN PRACTICE

広告クリエイティブでの活用

📊 売れてるCRの共通点を見つける

200本の動画をベクトル化して「勝ちパターン」を発見

1

データを集める

売れてる動画200本 + 売れてない動画100本を用意する

2

全部ベクトル化する

各動画の台本・コピーを数値の配列に変換する

3

「勝ちゾーン」が見える

ベクトル空間上で、売れてる動画が集まっている場所 = 「勝ちゾーン」

4

新しいCRを「勝ちゾーン」の近くに生成する

勝ちパターンに近い、でもコピーではない新しいCRが生まれる

類似度スコア意味アクション
0.8以上ほぼ同じ差別化が足りない。角度を変える
0.7〜0.8同じカテゴリ良い方向性。微調整で使える
0.5〜0.7うっすら関連新しい切り口として検討価値あり
0.5未満無関係方向が違う。やり直し
EMBEDDING V2

エンベディング v2 — テキストだけじゃなくなった

従来はテキストだけだったベクトル化が、画像・動画・音声にも拡張されました。

エンベディングの進化

v1（従来）: テキスト → ベクトル

v2（最新）: テキスト + 画像 + 動画 + 音声 → 同じベクトル空間
 「おはようの文字」と「おはようと言っている動画」が
 同じ空間上で近くにプロットされる
これが何を変えるのか

今まで: テキストはテキスト同士でしか比較できなかった。
これから: 「この動画に一番合うコピーはどれ？」が計算で出せる。動画の雰囲気とコピーのトーンの「相性」を、数値で測れるようになる。

🧲 ベクトル検索を体験しよう

Claude Codeを使って、実際にベクトルの「意味の近さ」を体感するハンズオン。

ゴール

3つのテキストペアの「意味的な近さ」をAIに分析させ、ベクトル検索の概念を体感する

  1. Claude Codeで以下を実行:
    以下の3組のテキストペアについて、
    「意味的な近さ」を0.0〜1.0で推定してください。

    # ペアA
    テキスト1: 「膝が痛くて階段がつらい」
    テキスト2: 「関節のこわばりで歩行に不安」

    # ペアB
    テキスト1: 「膝が痛くて階段がつらい」
    テキスト2: 「今日はいい天気ですね」

    # ペアC
    テキスト1: 「初回半額でお試し」
    テキスト2: 「まずはワンコインで体験」

    各ペアの推定スコアと理由を説明して。
  2. 結果を見て「キーワードは違うけど意味は近い」ケースを確認する
  3. 自分の業務に関連するテキストペアを3組作って同じ分析を依頼する
  4. 「この分析を自動で行い、0.7以上のものだけ採用するフローを設計して」と追加指示
所要時間: 約25分
成果物

ベクトル検索の概念理解 + 自分の業務でのベクトル活用アイデア。
