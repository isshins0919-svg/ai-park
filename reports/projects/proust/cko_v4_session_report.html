📖用語解説

今回のセッションで登場した専門用語を全部わかりやすく解説。

mCVR（エムシーブイアール）
記事経由コンバージョン率。記事LPを見た人が最終的に買った割合。
mCVR = CV数 ÷ PV数 × 100

CTRだけだと「クリックしたけど買わない」人を拾ってしまうのでmCVRを最重要指標にしている。
王者: PV 1972 / CV 15件 → mCVR 0.76%
敗者: PV 1504 / CV 3件 → mCVR 0.20%
funnel_position（ファネルポジション）
記事LPの中で「今どこを読んでいるか」を表す位置情報。購買心理の流れに沿って7段階に分類する。
opening → empathy → concept → mechanism → proof → authority → cta
20要素スコアリング（A〜E層）
記事LPの各チャンクを「どの心理に働きかけているか」で20要素×0〜1点で評価する仕組み。

• A層: 共感・恐怖・後悔・不安（痛みへの共鳴）
• B層: 権威・社会証明・データ・透明性（信頼の構築）
• C層: 因果・独自メカニズム・差別化・反論処理（論理の証明）
• D層: 変化・憧れ・シナリオ・希少性（欲求の喚起）
• E層: 緊急性・オファー・リスク逆転・CTA明確さ（行動の促進）
chunk（チャンク）
記事LPをAIが処理しやすい単位に分割したテキストの塊。画像・テキスト・動画をそれぞれ1チャンクとして扱う。

チャンク数が多い = 記事が長い・情報量が多いことを意味する。
王者: 55チャンク（text28 / image27 / video 0）
敗者: 108チャンク（text161 / image33 / video29）
embedding（エンベディング）
テキストを数値のベクトル（3072個の数字の羅列）に変換すること。
意味が似ているチャンク同士は数値的に近くなるため「意味検索」が可能になる。

kiji-ragでは全チャンクをembeddingして「王者LPの勝ちパターンを意味で検索」できるようにしている。
「ワキガの悩み」→ [0.023, -0.14, 0.87, ...] (3072次元)
swing_mode（スイングモード）
改善の2案（A案/B案）をどのくらいの変化幅で作るかの方針。

• A = small: 両案とも小改修（局所的な修正）
• B = medium: 両案とも中規模改修
• C = 幅最大: A案=small / B案=large（幅広く試す）
• B-modified ★今回: 両案とも「削る」方向のmedium改修（新発見から生まれたモード）
kiji-rag（記事ラグ）
記事LP専用のVector RAG（Retrieval-Augmented Generation）システム。
過去の王者記事LPをembeddingで蓄積し、「今書こうとしているチャンクに似た勝ちパターン」を瞬時に参照できる。

商品LP（EC-ForceのLP）とは完全に分離した別DBで管理。
D3_scenario（ディースリーシナリオ）
20要素のうちD層の3番目「シナリオ = 読者が自分を主人公として物語に投影できるか」を評価する指標。

王者記事LPは冒頭で「行きつけの服屋で...」という具体的シナリオを入れてD3を1.2点獲得。
敗者は0点 → 読者が物語の主人公になれないまま終わる。
王者 opening D3_scenario: 1.2点
敗者 opening D3_scenario: 0.0点 ← 最重要病因
SquadBeyond MCP
記事LPのABテスト配信プラットフォーム「SquadBeyond」をClaude Codeから直接操作できるツール。

今回はlistAbTests / getBranchOperationReportsで実績データを取得し、配信中記事のmCVRを確認した。
情報過剰型敗者（逆説的発見）
「要素をたくさん入れれば勝てる」は間違い、という今回発見した逆説。

敗者はB/C/D/E層スコア合計で王者を上回るのに負ける。原因は「情報の過剰・冗長化」による読者の疲弊。
20要素ルーブリックの足し算最適化は逆効果というのがCKO v4.0の核心知見。

💊処方箋方針（次フェーズへの橋渡し）

診断書が出した swing_mode と推奨カード。実際の書き換えは kiji-prescriber が行う。

B
swing_mode: B-modified（新発見から生まれたモード）
従来のA/B/Cモードでは「欠損深さ」で判定していたが、今回の病因は「過剰さ」。
両案ともに「削る」方向のmedium改修が必要。削り方の幅で差を出す。
📋 A案: medium-削減型
① opening を再構築（D3_scenario 1.2点投入）
② cta を 25 → 8チャンクに圧縮
③ mechanism を 32 → 20チャンクに整理
既存構造を保持しつつ濃度を倍に
🔄 B案: medium-再構築型
① opening を物語型に完全リプレース（『冬の○○な日に...』）
② mechanism + cta を半分以下に圧縮
③ 削いた分 authority に医師ドラマを厚く積む
情報量1/2・感情濃度2倍

🔑 CKO v4.0 核心知見（今回確立）

「20要素ルーブリックでスコア合計が高い記事LPが勝つ」という前提は間違い。
情報過剰・冗長型のLPはスコアで勝っても mCVR で負ける。

正解は「必要な要素を必要な位置に適量入れる」こと。openingのD3_scenario（自己投影シナリオ）1チャンクが、mechanismの余分な10チャンクより価値が高い場合がある。
この逆説的発見が残り7体のagent設計指針を全て変える。
