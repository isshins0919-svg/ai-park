## 2. タイミング設計の鉄則

### 「テロップは発話の少し前から見せる、でも次の発話が始まるまで消さない」

```python
# TIMING_OFFSET = -0.35s（350ms前出し）
# between(adj_start, raw_end) の意味:
#   adj_start = Whisper検出時刻 - 0.35s  → 先出し
#   raw_end   = 次シーンのWhisper検出時刻（オフセットなし）→ ちょうど消える

# NG例（過去のミス）:
between(t_adj, t_next_adj)  # 両方オフセット → 「これ」の途中でテロップが変わる

# OK例:
between(t_adj, t_next_raw)  # 開始は早め、終了は実際の発話まで保持
```

### なぜ重要か
- 「子どものご飯にこれ」の「これ」が 0.82s に発話
- 次シーン「こっそり〜」のadj_start = 0.85s → 0.03s差でかぶる
- raw_endを使えば 1.20s まで前シーンのテロップが維持される → ズレなし

### Whisperマッチングの注意点
- 長いテキスト（15文字以上）は後半の単語にマッチしやすい
- search_afterを適切に更新して前シーン以前にマッチしないように制御
- スコア閾値: 0.45（下げすぎると誤マッチ）

## 3. 映像レイアウトのルール

### 4:3 vs 16:9 の判断
- **人物ショット（整体師・医師・話者中心）→ 4:3**
  理由: 縦に広いので顔〜上半身が自然に収まる、参考動画準拠
- **風景・商品・B-roll → 16:9**
  理由: 横の情報量を活かす

### ズーム設計
```
ZOOM_FACTOR = 1.15  # 人物が小さく見えるとき増やす（最大1.5程度）
CROP_Y_SHIFT = 30   # 上方シフト（顔を上部に保持）正の値ほど上寄り
```

### FV冒頭黒フレーム対策
```python
# blackdetectで自動検出してスキップ
# pic_th=0.90: 90%以上黒いフレーム
# d=0.05: 0.05秒以上続く黒区間を検出
skip_t = get_first_non_black_time(clip_path)
```
- 検出できない場合は手動で `-ss 0.3` などを試す
- FVクリップが.movの場合、カメラ起動直後の暗いフレームが多い

## 4. テロップ改行の設計原則

### wrap_telop_smart() の考え方
「文字数で単純に切ってはいけない。意味の区切りで切る」

```
NG: 「別人かってぐらいぐんぐん伸|びた」（13文字で機械的に切る）
OK: 「別人かってぐらい」/「ぐんぐん伸びた」（助詞「い」で自然分割）
```

分割基準（優先順）:
1. `\n` が明示的に入っていれば必ずそこで改行
2. 全体長さの中央±4文字の範囲で助詞・読点を探す
3. フォールバック: max_charsで単純分割

### 分割候補文字（SPLIT_CHARS）
```
が、の、に、を、は、も、で、へ、と、よ、り、か、ら、ま、で、
つ、い、て、も、な、ら、、、。、！、？
```

### CSVで手動改行する場合
セルに改行を入れるか `\n` 記法を使う（将来対応予定）

## 5. 音声設計の鉄則

### 「ボディ部分は絶対に切り刻まない」
- body.mov は1本丸ごとFFmpeg処理
- テロップ切替は `enable='between(t,...)'` のoverlay方式
- 音声ストリームには一切触れない → `map 0:a` をそのまま通す

```python
# NG（過去のミス）: シーンごとにFFmpegを呼んで結合
for scene in body_scenes:
    ffmpeg(-ss start, -t duration, ...)  # 音声が継ぎ目でブツ切れ

# OK: 1本のFFmpegコマンドでtimed overlay
ffmpeg -i body.mov -loop 1 -i scene1.png -loop 1 -i scene2.png \
  -filter_complex "[0:v]scale...[base];[base][1]overlay=enable='between(t,0,1.2)'[s1];..."
```

## 6. 注釈テキストのoverflow対策

### wrap_text_by_width()
- ピクセル幅基準で折り返す（文字数ではなく実際の描画幅）
- ANNO_MAX_W = OUT_W - 24（左右12pxマージン）
- VIDEO_Y(520px)を超えそうになったらクランプして終了

### 注釈フォント
- サイズ: 26px（W3 regular）
- 行間: 32px
- 色: fill=(255,255,255,200)（少し透明で主張しすぎない）

## 7. FFmpegコマンド設計パターン

### パターンA: FVシーン（単一クリップ + 静的overlay）
```
ffmpeg -ss {skip_t} -i fv.mov -loop 1 -i overlay.png
  -filter_complex "[0:v]{vf}[base];[base][1:v]overlay=0:0[out]"
  -map [out] -map 0:a -c:v libx264 -crf 18 -af loudnorm -shortest
```

### パターンB: ボディ（連続音声 + timed overlay）
```
ffmpeg -i body.mov -loop 1 -i static.png -loop 1 -i s0.png ... -loop 1 -i sN.png
  -filter_complex
    "[0:v]{vf}[base];
     [base][1:v]overlay=0:0[s0];
     [s0][2:v]overlay=0:0:enable='between(t,{adj0},{raw1})'[s1];
     [s1][3:v]overlay=0:0:enable='between(t,{adj1},{raw2})'[s2];
     ..."
  -map [sN] -map 0:a -c:v libx264 -crf 18 -af loudnorm -t {body_dur}
```

### パターンC: FV + ボディ結合
```
ffmpeg -f concat -safe 0 -i concat.txt
  -c:v libx264 -crf 18 -c:a aac -ar 44100 -ac 2
```
※ `-c copy` は非推奨（コーデックの差異でエラーになりやすい）

## 8. アプリ化に向けた「ノーミス設計」仕様書

### 入力仕様（CSVフォーマット）
```
列: [No, テロップテキスト, クリップファイル名, 注記]
FV行: "1-1", "1-2" ... "1-9"
Body行: "2", "3" ... の整数
```

### 将来的な入力UI要件
- テロップテキストで `|` または `<br>` を手動改行に使えること
- タイムスタンプ列（手動オーバーライド用）: 空欄ならWhisper自動
- ズーム倍率・上シフト量をFV/bodyで個別指定できること

### 品質チェックリスト（自動化すべき項目）
- [ ] 冒頭黒フレーム: blackdetectで検出・報告
- [ ] テロップ改行の意味的妥当性: 単語をまたぐ分割を検出・警告
- [ ] 音量レベル: loudnorm後に -14±2 LUFS に収まっているか
- [ ] テロップと発話のズレ: between区間と実際の発話時刻の差を報告
- [ ] 注釈overflow: VIDEO_Yを超えていないか

### Whisperの精度向上策
- `word_timestamps=True` は必須
- モデル: `mlx-community/whisper-large-v3-turbo`（速度と精度のバランス）
- 長いテキストはマッチング精度が落ちるため、最初の5〜8文字でマッチングする
- search_afterで前シーンより後のwordのみを対象にする

## 10. カントク君の判断基準

### テロップタイミングで迷ったとき
「音声と同時か少し前から出す。次の音声が始まるまで消さない」

### 改行で迷ったとき
「声に出したときに自然に息継ぎする場所で切る」

### レイアウトで迷ったとき
「参考動画を計測してから作る。直感で決めない」

### 新しいクライアントの動画を作るとき
1. 参考動画を必ず `ffprobe` で解像度・音量を実測
2. `clients/` のクライアントファイルを確認
3. このルールブックと参考動画を照合してズレがないか確認してから着手
