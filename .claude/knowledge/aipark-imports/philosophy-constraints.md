## 使い方

以下のチェックリストを、CR仮説を生成する全てのスキル（banner-park / shortad-park / 記事LP-park / kiji-cko等）で使用する。

```
Step 1: 仮説を生成
Step 2: 本チェックリストに照合
Step 3: FAIL項目があれば REJECT → 代替を即生成
Step 4: PASS時のみ次のフェーズへ進む
```

## 3つの魂（全仮説に適用）

### 1. 愛

- [ ] **相手を否定していないか？** 「まだ人力で営業？」「テレアポはもう古い」等のNG角度になっていないか
- [ ] **人とAIの掛け算になっているか？** AIが人間を置き換えるメッセージではなく、人間がより高い次元へ行ける構造か
- [ ] **相手の強みを認めた上で提案しているか？** 弱みを突くのではなく強みを拡張する

### 2. 真に偉大か

- [ ] **安易な訴求に逃げていないか？** 割引/限定/損失回避だけに頼っていないか
- [ ] **1年後も成立するメッセージか？** トレンド依存の一過性コピーではないか
- [ ] **コンセプトが末端まで浸透しているか？** CR全体（コピー+ビジュアル+仮説）でコンセプトが貫通しているか

### 3. 全て可能だとしたら

- [ ] **既知のパターンの焼き直しではないか？** RAGの勝ちパターンを参考にしつつも、未探索の可能性に挑戦しているか
- [ ] **プロダクトの可能性を最大限に描いているか？** 機能説明ではなく、その機能が生み出す未来の姿を表現しているか
- [ ] **技術的に実現できたはずの機会を逃していないか？** AGIの力を最大限に使い切っているか

## ビジュアル制約

- [ ] **プロダクト実素材を使っているか？** 汎用AI生成画像ではなく、案件固有の実素材（商品画像/ブランドロゴ/UI等）
- [ ] **keyVisual の differentiator と整合しているか？** strategy.json or clients/ の keyVisual セクションの構図・色彩・ムードに沿っているか
- [ ] **余白に意図があるか？** 空白ではなく「意図的な呼吸」としての余白。情報密度が適切か

---

## コピー制約

- [ ] **層別の権威性バッジを正しく使っているか？** 対象層に対応するバッジ・証拠（超潜在/潜在/準顕在/顕在で異なる）
- [ ] **一次情報と二次情報が分離されているか？**
  - [一次] = DPro / Meta API 実データ / 商品LP取得値
  - [二次] = Opus 推論 / Gemini 分析
  - [三次] = WebSearch / 推測
- [ ] **数字の根拠が示されているか？** 「X万人が選んだ」の X は実データか

---

## 判断基準の最新パターン（VOYAGE版の派生記述）

- **データが全てを決める**: 感覚で止めるな。データだけが退場を決める
- **時間は代理変数、本質はデータの状態**: 10日経過ではなく、freq > 3.0 or CTR 20%低下 等のデータで判断
- **完遂してから報告**: 途中で「いい？」と聞くな。完走して結果で返す
- **AGI推論に賭ける**: Pythonの if文 で判断できることはPython。因果推論 / 仮説生成 / 批判的検証 は Opus
- **本質で外してOK**: ルールを機械的に適用するな。本質的に考えて、ルールから外す勇気を持て

## 失敗時のアクション

1. 具体的にどのチェック項目でFAILしたかを記録
2. 仮説をREJECT（廃棄 or 再設計）
3. `losing_dna` に失敗パターンを構造化して蓄積
4. 次回同種仮説生成時にフィルタ
