# sales_elements — 20要素スコアリングルーブリック

記事LPチャンクに付与する **セールスライティング20要素**（5層 × 各4要素）の定義と判定基準。

各要素は **0.0〜1.0 の連続値**。LLM判定（Claude Haiku or Sonnet）で採点。

---

## 階層構造（5層 × 4要素 = 20）

| 層 | 層の役割 | 要素ID |
|---|---|---|
| **A. 共感層 (Empathy)** | 読者の現状不満・痛み・不安を引き出す | A1〜A4 |
| **B. 信頼層 (Trust)** | 権威・証拠・透明性で信用を作る | B1〜B4 |
| **C. 論理層 (Logic)** | 因果と機序で「なぜ効くか」を納得させる | C1〜C4 |
| **D. 欲求層 (Desire)** | 理想像・快の予感で欲しくさせる | D1〜D4 |
| **E. 行動層 (Action)** | 購入ボタンを押させる直接要素 | E1〜E4 |

A→B→C→D→E は**記事LPの自然なファネル順**。各チャンクで最大スコア層 = `dominant_layer`。

---

## 20要素 定義

### A. 共感層 (Empathy)

| ID | 要素 | 定義 | 強い例 (1.0寄り) | 弱い例 (0.2-0.4) |
|---|---|---|---|---|
| A1 | pain_empathy | 読者の痛みを具体的に言語化 | 「いちご鼻が汚すぎ」「逆に悪化した気さえする」 | 「毛穴が気になる方へ」 |
| A2 | fear_appeal | 放置すると悪化する/手遅れになる恐怖 | 「一生この汚い肌なの…」「放置すると取り返しがつかない」 | 「ケアが大切」 |
| A3 | regret_avoidance | 今動かない・買わないと後悔する | 「今スルーすると損」「9,000円損した」 | 単なる促し |
| A4 | anxiety_trigger | 時間的・機会的な焦り | 「明日売り切れる危険」「いつ終了するか」 | 軽い呼びかけ |

### B. 信頼層 (Trust)

| ID | 要素 | 定義 | 強い例 | 弱い例 |
|---|---|---|---|---|
| B1 | authority | 監修者・専門家・受賞・メディア掲載 | 「MEGUMI監修」「日本一売れた美容本」「LDK A評価」「美容皮膚科」 | 「専門家推奨」（誰か不明） |
| B2 | social_proof | 販売数・口コミ数・ランキング・利用者数 | 「楽天1位」「発売2日で100媒体」「POPUP大盛況」 | 「人気」（数なし） |
| B3 | data_evidence | 具体的な数値・実験・科学根拠 | 「美容成分90%配合」「毛穴年齢19歳」「1000種類試した」 | 「成分豊富」 |
| B4 | transparency | 開発秘話・失敗談・裏側・個人史 | 「20代の頃は毛穴に悩んでた」「10年で1000種類試した」 | 一般論的ブランドストーリー |

### C. 論理層 (Logic)

| ID | 要素 | 定義 | 強い例 | 弱い例 |
|---|---|---|---|---|
| C1 | causality | 問題の原因・メカニズムの解明 | 「オイルは美肌バリアまで洗い流す→皮脂過剰分泌」 | 「毛穴が開くから」 |
| C2 | unique_mechanism | 独自の仕組み・世界初/日本初 | 「日本初2層式クレンジング」「角栓溶解法」「独自ナノ粒子」 | 「こだわりの製法」 |
| C3 | differentiation | 既存品との明確な違い・カテゴリ対比 | 「オイルは逆効果、ミルクは洗い残す、本品は両立」 | 「他と違う」 |
| C4 | objection_handling | 想定反論の先回り処理 | 「定期だけどいつでも解約可」「効果なければ全額返金」 | 反論未対処 |

### D. 欲求層 (Desire)

| ID | 要素 | 定義 | 強い例 | 弱い例 |
|---|---|---|---|---|
| D1 | transformation | ビフォーアフター・変化の描写 | 「毛穴目立つ肌→ぷるつや卵肌」「えぇぇ！毛穴見えない」 | 「効果あり」 |
| D2 | aspiration | 理想未来・憧れ像の提示 | 「すっぴん美人」「MEGUMIさんみたいに」「自信ついた」 | 「綺麗になる」 |
| D3 | scenario | 使用シーン・生活場面の具体化 | 「すっぴんで過ごせる」「仕事・家事・育児のママ」 | 「毎日使える」 |
| D4 | scarcity | 希少性・所有欲・限定 | 「完売」「入手困難」「限定300セット」「YouTube見た人だけ」 | なし |

### E. 行動層 (Action)

| ID | 要素 | 定義 | 強い例 | 弱い例 |
|---|---|---|---|---|
| E1 | urgency | 時間的緊急性（今動け） | 「今のうちに」「明日完売」「本日限定」 | なし |
| E2 | offer_appeal | 価格の魅力・割引・特典 | 「69%OFF」「1,980円」「無料特典3点」 | 定価のみ |
| E3 | risk_reversal | 返金・解約・保証 | 「全額返金保証」「いつでも解約可」 | なし |
| E4 | cta_clarity | 購入動線の明瞭性・ボタン存在 | 「公式サイトはこちら」ボタン、購入URL | 動線曖昧 |

## スコアリング時の判定プロンプト（LLM用）

```
以下のチャンク内容を読んで、20要素 (A1〜E4) 各々について
0.0〜1.0 のスコアを出力してください。

判定基準:
- 0.0: 要素が全く含まれない
- 0.3: 要素の片鱗だけある（明示的でない）
- 0.5: 要素がはっきり含まれる（標準レベル）
- 0.7: 要素が強く訴求されている
- 1.0: 要素がチャンクの核になっている

注意:
- 同じチャンクで複数要素が高スコアになるのは正常
- 画像ブロックは周辺テキスト・caption・alt から判定
- 「要素の強さ」を測るのであって、チャンク全体の良し悪しではない

出力フォーマット: JSON
{
  "A1_pain_empathy": 0.8,
  "A2_fear_appeal": 0.3,
  ...
  "E4_cta_clarity": 0.0
}
```

## 派生指標（チャンクごとに自動計算）

- `dominant_layer` = max(A_avg, B_avg, C_avg, D_avg, E_avg) の層
- `dominant_elements` = 20要素の上位3つ
- `layer_scores` = {A: A1..A4の平均, B: ..., E: ...}
- `total_intensity` = 20要素の単純平均（チャンクの「訴求密度」指標）

## 既存 kijiスコア との対応

| 20要素側 | kiji系スコア側 |
|---|---|
| A層（A1〜A4） | kiji-hook-crew（悩み解像度・感情温度） |
| B層（B1〜B4） | kiji-trust-crew（権威×数値×口コミ） |
| C層（C1〜C4） | kiji-arc-crew（新認識・中だるみ） |
| D層（D1〜D4） | kiji-arc-crew（V字アーク）+ kiji-compass |
| E層（E1〜E4） | kiji-cta-crew + kiji-offer-crew |

20要素 = **チャンク粒度**、kiji系 = **記事全体粒度**。両立する設計。

将来、`B層平均 vs kiji-trust_score` の相関を取れば、20要素判定の妥当性を検証できる。
