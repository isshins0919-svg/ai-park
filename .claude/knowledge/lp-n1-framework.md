# LP N1フレーム — 買う瞬間の動機軸

> **N1は「悩み軸」ではなく「買う瞬間の動機軸」で立てる**
> ameru案件（2026-04-20）で確立されたパク流N1設計の核。
> 広ターゲットLPでも、実際にCVする人は特定像に収束する。
> N1特定 = 射程を狭めるではなく、コピー解像度を上げる道具。

---

## 🚨 絶対守る原則

### 1. 悩み軸で立てるな
❌ 悪い例：「デジタル疲労に苦しむ30代女性」「編み物で何度も挫折した人」
→ 悩み軸で立てると、**ネガ起点のコピー**になりLP全体が暗くなる。射程も狭くなりすぎる。

✅ 良い例：「推しキャラの新キット告知を見た瞬間、"買うより編みたい"と思う人」
→ **買う瞬間のトリガー+ポジ動機**。コピーが肯定形で書ける。射程は自然に広がる。

### 2. 悩みで分けるな、動機で分けるな
同じ商品でも「なぜ買うか」が違う → **別LPで別N1**を立てる。

ameru例：
| LP | N1軸（動機） | 同じ「挫折経験」ではなく別軸 |
|---|---|---|
| 原本 | 推し×所有欲 | 推しキャラを自分の手で迎えたい |
| A | 自己効力感 | "できる自分"の証拠が欲しい |
| B | デジタル疲労回避 | 何も考えない時間が欲しい |
| C | コレクター欲 | 全キャラコンプを自作で達成したい |

→ **1商品=1N1ではない**。LPバリエごとにN1を変える。

### 3. IPフック商品は"動機"で狭く切れ
「挫折経験×デジタル疲労×推し活」の三位一体のような**射程の狭すぎるN1**は、IPフックLP文脈と衝突する。

買う瞬間の動機は**1本だけ**に絞る。抵抗は3つまで。

---

## 🗺️ N1カードの型（これをコピペして埋める）

```markdown
### [LP名]: [動機軸1行]

| 項目 | 内容 |
|---|---|
| **N1像** | [年代・性別・職業・商品との接点を1文で] |
| **トリガー瞬間** | [いつ・どこで・何を見て「欲しい」と思うか] |
| **コア動機** | [ポジ側の欲望。"〇〇したい"で書く] |
| **3大抵抗** | ①[購入直前の躊躇1] ②[躊躇2] ③[躊躇3] |
| **LPコピー軸** | [1行で表現するメインメッセージ] |
```

### 埋め方のコツ

- **N1像**：解像度はざっくりで良い。年代+職業カテゴリ+商品接点があればOK
- **トリガー瞬間**：ここが一番大事。「いつ」「どこで」「何を見て」の3W+「何を思った」を具体的に
- **コア動機**：必ず肯定形・ポジ側で書く。「〇〇がつらい」ではなく「〇〇したい」
- **3大抵抗**：価格/時間/自己不信/リスクなど。購入直前の躊躇を想像する
- **LPコピー軸**：FVに置けるレベルの1行。最終的にこれが刺さるかで勝負

## 🔄 使うタイミング

1. クライアント着手前に `.claude/clients/[name]/` 読む
2. 本スキルを参照してN1カードを**1原本+3バリエ分（合計4枚）**作成
3. ユーザーに提示 → 動機軸でGO判断もらう
4. 各LPのFVコピー・セクション構成はN1カードから逆算

---

## ❌ よくあるN1設計ミス

| ミス | なぜダメ | 対処 |
|---|---|---|
| 悩み軸で立てる | LPがネガ起点になる | 買う瞬間の動機に置き換える |
| 1商品1N1で作る | バリエの差別化が不明瞭 | LPごとに別N1を立てる |
| 三位一体N1 | 射程狭すぎ、複数軸で衝突 | 1動機軸に絞る |
| 抵抗が4つ以上 | コピーで全部打ち消せない | 3つまでに絞る |
| トリガーが抽象的 | コピーが弱くなる | 3W+感情で具体化 |
