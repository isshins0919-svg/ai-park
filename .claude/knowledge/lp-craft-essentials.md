## 🎯 LPのジャンルを最初に確定する

LP制作に着手する前に必ず確定する。ジャンルを誤認すると評価軸も実装も全部ズレる。

| ジャンル | 例 | 構造的特徴 | 主軸 |
|---|---|---|---|
| **DR-LP（ダイレクトレスポンス）** | regina-clinic / kimie / laric / yuriiro / felissimo | HTML骨格＋画像セクション、CTA 5-10回反復、価格%OFF強調、FAQ豊富 | **CVR最大化** |
| **ブランドHP** | SHIRO / FUJIMI / Craftie | 余白、抑制、世界観優先、CTA控えめ | **LTV/ブランド資産** |
| **教育・案内系LP** | gpt55 / Design School系 | ハイブリッド、コース紹介中心 | **入会・無料相談** |
| **ECサイト商品ページ** | Amazon / 楽天 | テンプレート主導、画像連続 | **比較検討→購入** |

**着手前必須確認**:
1. このLPのジャンルは？
2. 参考LPの URL（3〜6本）は？
3. CVR優先かブランド優先か？
4. 既存素材（実写撮影）はあるか？

## 🏗 売れている DR-LP の構造原則（再現可能な型）

```
[1] HTMLヘッダー（ロゴ1回 + 電話）
[2] FV: 大画像背景 + HTML見出し + HTML CTAボタン + 価格バッジ
[3] 共感セクション: HTML箇条書き＋アイコン
[4] Only1: 画像3カット（ビフォー→中間→アフター）
[5] 数値証拠: HTMLで大数字
[6] ユーザーボイス: 顔写真 + 実名年齢 + ★★★★★ + コメント
[7] 権威: メディアロゴ（テレビ/雑誌/賞）
[8] 機能・効能: 図解画像 + HTMLリスト
[9] オファー（最重要）: HTMLテーブル + 画像5大特典
[10] 保証: 5バッジ
[11] FAQ: HTMLアコーディオン
[12] 最終CTA: 画像 + HTML巨大ボタン
[13] HTMLフッター: 特商法 + 規約
```

**鉄則**:
- ロゴ繰り返し: ヘッダー1回 + フッター1回 = 2回 が正解（V6は20回繰り返した = NG）
- CTA反復: 5〜7回
- 画像内焼き込み: メインキャッチ・%OFF・特典バナーのみ
- HTMLテキスト: ボディ・規約・成分・FAQ
- 法務テキストは必ずHTMLで（SEO・法規対応）

## ❌ V6 の失敗から学ぶ「やってはいけない」

### 1. 全画像LP（フルスクリーン10枚並列）
**症状**: 1024x1536画像10枚を縦並列、HTMLは imgタグ並列のみ
**結果**: 「LP」ではなく「広告10枚集」、独立広告のループに見える
**正解**: HTML骨格＋画像はセクション内素材

### 2. ロゴ・Section番号の全画像焼き込み
**症状**: V6は「ameru × らぶいーず®」+ 「Section X/10」を全10枚に焼き込み
**結果**: ノイズ増加、シリーズ感のつもりが「同じ広告ループ」感
**正解**: ロゴは画像外HTMLヘッダーで1回、Section番号はそもそも不要

### 3. 海外ブランド世界観の表層翻訳
**症状**: V6-WはThe Woobles英語コピー（"Sound familiar?" "Real wins."）をそのまま使用
**結果**: 日本人ターゲットに違和感、読みづらい外国語LP化
**正解**: 海外ブランドの「本質的価値」を日本語に翻訳。色・モチーフ・トーンは継承、コピーは現地化

### 4. 価格・FAQ・規約の画像焼き込み
**症状**: V6は価格表・FAQ・チェックリスト全部画像内
**結果**: A/Bテスト不可、SEO効かず、修正に再生成必要、特商法不備
**正解**: 価格・FAQ・規約はHTML（必須）

### 5. CTAボタンの不足
**症状**: V6は Sticky CTA 1個のみ
**結果**: スクロール途中の購買意欲を回収できない
**正解**: 5〜7箇所にCTAボタンを反復配置

## 🎨 海外ブランドLP「本質翻訳」の方法

「The Woobles」「Craftie Home Box」など海外/他社ブランドの世界観を導入するとき、**英語をそのまま使うのは表層的**。本質を抽出して日本語で翻訳する。

### Wooblesの本質要素（V7用）

| Wooblesの本質 | 表層翻訳（V6-Wの間違い） | 本質翻訳（V7の正解） |
|---|---|---|
| Fiero（達成感） | （翻訳できず英語） | 「『私にもできた』を、約束します」 |
| Pre-started piece | "Pre-started piece" | 「いちばん難しい1段目、もう編んであります」 |
| Beginner-friendly | "BEGINNER OK" | 「はじめてでも、絶対」 |
| Easy Peasy Yarn | "Easy Peasy Yarn" | 「割れない、ほつれない、初心者専用糸」 |
| Verified Buyer | "Verified Buyer" | 「★購入者の声」 |
| Cancel anytime | "Cancel anytime" | 「いつでも解約OK」 |

### 翻訳の鉄則
- **色**: そのまま継承可（Wooblesイエロー使ってOK）
- **モチーフ**: そのまま継承可（derpyキャラ造形、★レーティング）
- **トーン**: そのまま継承可（フレンドリー、エンパワーリング）
- **コピー**: 必ず日本語化（短英単語"Easy"等は混入可、文章は不可）
- **写真演出**: そのまま継承可（白背景、自然光、UGC風）

## 🧠 本質的なLP設計原則 10ヵ条

1. **構造はHTML、画像は素材** — 全画像LPはNG
2. **ロゴは2回（ヘッダー＋フッター）** — 画像内繰り返し禁止
3. **CTAは5〜7回HTMLで反復** — Sticky依存はNG
4. **価格/FAQ/規約はHTML必須** — 法規・SEO・A/Bテスト対応
5. **数値は仮想でも明示** — 「累計X名」「満足度Y%」がないLPは死ぬ
6. **顔写真ボイス必須** — 実名+年齢+具体エピソード
7. **権威は重層化** — メディア×専門家×アンバサダー×実績
8. **緊急性は数値化** — 「残り◯セット」「期間限定」を可変表示
9. **海外ブランドは本質翻訳** — 英語そのまま使用は表層的、必ず日本語化
10. **ジャンル先行確認** — DR/ブランド/教育/EC のどれかを最初に決める

## 🔄 AI×LP制作の正しいパイプライン（V7以降の標準）

```
[Step 1] ジャンル確定（DR/ブランド/教育/EC）
[Step 2] 参考LP収集（3〜6本、勝ちパターン）
[Step 3] embedding-2 でマルチモーダルベクトル化
[Step 4] 各セクションごとに参考画像 top-k retrieval
[Step 5] 素材生成（画像はキービジュアルのみ、ロゴ・コピー禁止）
[Step 6] HTML骨格を組み、画像を埋め込み
[Step 7] HTMLでCTA・価格・FAQ・規約を実装
[Step 8] 法規チェック（特商法・薬機法）
[Step 9] パフォーマンス計測（mCVR/LTV）
[Step 10] A/Bテスト（HTMLテキスト変更で素早く回す）
```

## 🏴‍☠️ 核心メッセージ

**「LPはAIで作れる。ただし、構造を間違えるとAIで作っても売れない。」**

V6の失敗が証明した: gpt-image-2の品質は本物。でも全画像LPの構造で組んだら、売れているDR-LPと真逆になる。
本質は「画像のクオリティ」じゃなく「LPの構造」。
構造原則はAI以前の DR-LPコピーライターの30年の知恵。それを継承した上で AI を使う。

V7はその「構造原則 × AIの画像力」の融合。

## ⚠️ 致命的な失敗パターン（一進さん指摘・必ず学ぶこと）

### 失敗1: 過去のアウトプットを「旧版」として無視する
- **ダメな例**: pattern_01_meeth_beauty.html を「単なる旧版」として無視した
- **本来やるべき**: 過去のアウトプットの **「良かった要素」を必ず抽出して継承する**
- **検証質問**: 「過去版に何が良かったか？それを失っていないか？」を常に問う
- **指摘日時**: 2026-04-26

### 失敗2: 参考LPが送られた時、何を参考にすべきか確認を取らない
- **ダメな例**: 6本のDR-LPを「構造（HTML骨格）の参考」と勝手に解釈
- **本来やるべき**: 「参考に何を取り込めばいい？文字？デザイン？構造？戦略？」と **必ず質問する**
- **検証質問**: 参考資料が来たら「どの要素を参考にしますか？」と即聞く
- **指摘日時**: 2026-04-26

### 失敗3: 既存素材を使わず、自分で代替生成しようとする
- **ダメな例**: ameru ロゴ・パッケージ・公式キャラが提供されてるのに、画像生成で「らしきもの」を新規作成
- **本来やるべき**: 素材が提供されている = **必ず使う**。/images/edits の参考画像入力で素材を強制的に取り込む
- **原則**: 素材提供は「これが大事です」のシグナル。代替生成は LP制作者として絶対やらない
- **指摘日時**: 2026-04-26

### 失敗4: IPファンに刺さる権威の本質誤認
- **ダメな例**: 「日本テレビ公式コラボ」で権威強化と誤解
- **本来やるべき**: IPファンに刺さる権威は **IPブランド本体（らぶいーずの公式ロゴ）**。背後のテレビ局じゃない
- **マーケター視点**: 「IP好きの人」=「キャラを愛してる人」。テレビ局を愛してるんじゃない
- **指摘日時**: 2026-04-26

### 失敗5: V6のビジュアル衝撃を「画像LPだから問題」と過剰反応
- **ダメな例**: 一進さんの「ロゴ重複問題」指摘を「画像LP構造そのものの否定」に拡大解釈
- **本来やるべき**: 指摘の **核心だけ** 解決する。構造を勝手に変えない
- **検証質問**: ユーザー指摘を受けたら「これは部分修正？それとも全面リプレイス？」を確認する
- **指摘日時**: 2026-04-26
