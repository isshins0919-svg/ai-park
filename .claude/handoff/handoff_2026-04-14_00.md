## 次セッションでやること 🚀

### 1. GCP Service Account 作成（5分）
```
GCPコンソール (console.cloud.google.com)
→ プロジェクト: claude-x-api
→ IAMと管理 → サービスアカウント → 作成
  名前: kosuri-sheets
  ロール: 不要（スプレッドシートへのアクセスはシート共有で管理）
→ キー → JSONキーを作成 → ダウンロード
```

### 2. スプレッドシートにService Accountを共有
```
ダウンロードしたJSONの "client_email" をコピー
（例: kosuri-sheets@claude-x-api.iam.gserviceaccount.com）
→ スプレッドシートの「共有」から編集者として追加
```

### 3. ローカルでSheetsテスト（任意）
```bash
cp ~/Downloads/[ダウンロードしたJSON].json video-ai/fv_studio/service_account.json
python3 video-ai/fv_studio/app.py
# ブラウザ → 管理アイコン → Sheetsセットアップ → シートが作られれば成功
```

### 4. Cloud Run デプロイ（GCP課金が承認され次第）
```bash
cd ~/Desktop/一進VOYAGE号
bash video-ai/fv_studio/deploy.sh
```

### 5. Cloud Run に環境変数を設定
```
Cloud Run → kosuri-studio → 新しいリビジョンを編集
→ 環境変数を追加:
  GOOGLE_SERVICE_ACCOUNT_JSON = (JSONファイルの中身を丸ごとコピペ)
```
または deploy.sh に追加:
```bash
--set-env-vars="GOOGLE_SERVICE_ACCOUNT_JSON=$(cat video-ai/fv_studio/service_account.json)"
```

### 6. チームへURL共有
```
gcloud run services describe kosuri-studio --region asia-northeast1 --format="value(status.url)"
```
→ このURLを `@cartahd.com` の全員にSlackで共有
