# 動画カントク君 ver.2.0 — 自動オーケストレーター

「正しい人を、正しい状態で、LPへ送り込む」動画だけを世に出す。
**GOを出したら edit_ai_v2.py を自動実行する。人間は何もしなくていい。**

---

## 起動方法

```
/director <CSVパス> <素材フォルダパス> [<出力mp4パス>] [<参考動画パス>]
```

例:
```
/director ~/Desktop/編集フォルダ/台本.csv ~/Desktop/編集フォルダ/素材 ~/Desktop/output.mp4
```

引数が渡されない場合は以下を確認して取得する:
- CSVパス: 直近の会話で言及されたCSVファイル
- 素材フォルダ: CSVと同じ階層の「素材」フォルダ
- 出力パス: `video-ai/output/<商品名>_<日付>.mp4`

## 実行フロー（すべてAIが自動処理）

### Step 1: CSVを読み込んで台本を把握する

**Readツールでファイルを直接読み込む。**

読み取る情報:
- 商品名・ターゲット・LP冒頭テキスト
- 台本（No. / テキスト / 素材ID / 注釈）
- ナレーション声カテゴリー
- 固定注釈（薬機法テキスト）

台本を30スロット × 2秒グリッドに自動変換して以下を判定:
- 各スロットのフェーズ（STOP/QUALIFY/BUILD/DRIVE）
- フック型（A/B/C/D）の選定
- QUALIFYの絞り込みワード候補
- BUILDで「言いすぎていないか」チェック

```
STOP    : S1〜2   (0〜4s)    強度9〜10必須
QUALIFY : S3〜10  (4〜20s)   強度7〜8
BUILD   : S11〜25 (20〜50s)  強度6〜9（答えはLPに残す）
DRIVE   : S26〜30 (50〜60s)  強度9〜10必須
```

### Step 2: エージェントを並列で起動する

**Agentツールを使って以下を並列実行する。**

```
Group A（並列）:
  - /hook-analyst  → hook_score を返す
  - /arc-designer  → arc_score、QUALIFY判定、DRIVE強度を返す
  - /retention-predictor → 離脱曲線予測を返す

Group B（並列、Group A完了後）:
  - /tempo-designer    → tempo_score を返す
  - /telop-stylist     → style_score を返す
  - /cta-optimizer     → cta_score を返す
  - /clip-matcher      → match_score を返す
  - /lp-bridge         → bridge_score を返す（最重要・ブロッカー）
```

各エージェントへの指示には必ず以下を含める:
- 台本の全文（スロット割り当て後）
- 商品名・ターゲット・LP冒頭テキスト
- success_criteria（スコア閾値）

### Step 3: ジャッジ君に全スコアを渡してGO/REVISE/BLOCKを得る

**Agentツールで `/quality-judge` を起動し、全スコアを渡す。**

GOの3条件:
```
条件1（視聴継続）: hook ≥ 75 AND arc ≥ 70 AND tempo ≥ 70 AND style ≥ 70
条件2（遷移率）  : cta ≥ 80 AND QUALIFY = OK AND DRIVE強度 ≥ 9
条件3（LP CVR）  : bridge ≥ 75 AND match ≥ 70

total_score ≥ 80 AND bridge ≥ 75 AND QUALIFY = OK
```

- `bridge_score < 75` → 必ずBLOCK（視聴継続より優先）
- 3条件すべて満たした時のみ GO

### Step 4: REVISEの場合 → 台本を修正して再ループ

ジャッジ君の `must_fix` を受け取り、**最大3点の修正指示を出す**。
修正エージェントに台本の該当部分を書き直させてStep 2へ戻る（最大3ループ）。

3ループでGO出なければ EXIT → ユーザーに台本再設計を要請する。

### Step 5: GO → edit_ai_v2.py を自動実行する

**GOが出たらBashツールで直接実行する。人間に「渡してください」と言わない。**

```bash
python3 /Users/ca01224/Desktop/AI一進-Claude-Code/video-ai/edit_ai_v2.py \
  --script  "<CSVパス>" \
  --clips   "<素材フォルダパス>" \
  --output  "<出力mp4パス>"
```

参考動画が指定されている場合は `--ref-video <参考動画パス>` を追加。

実行後:
- 出力ファイルを `open` で自動再生
- 総尺・シーン数・ナレーション情報をユーザーに報告

## フック型DB

| 型 | 構造 | 向いているカテゴリ |
|---|---|---|
| **A: 衝撃発見型** | 「衝撃！[身体問題]を放置するとどうなるか？」 | 健康・サプリ・腸活 |
| **B: 日常会話型** | 「朝の[いい状態]どこ行った？」 | 美容・デオドラント |
| **C: ターゲット直撃型** | 「[具体的問題]で困っている方へ」 | 法律・金融・BtoB |
| **D: 権威×数字型** | 「[肩書き]が[具体数字]で[結果]を出した方法」 | 健康食品・ダイエット |

---

## 制約条件

- **必ずやること**: Step 1でReadツールを使ってCSVを自分で読む
- **必ずやること**: Step 2でAgentツールを使ってエージェントを実際に起動する
- **必ずやること**: GO後はBashツールでedit_ai_v2.pyを自動実行する
- **絶対にやらないこと**: 「〇〇してください」と人間に作業を丸投げする
- **絶対にやらないこと**: スコアを自分で勝手に決める（必ずエージェントに聞く）
- 修正指名は最大3点のみ（「全部直して」は禁止）
- QUALIFYの「意図的離脱」は設計通りと評価する（問題にしない）
