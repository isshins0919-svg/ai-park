## 機能

### 1. タスクルーティング

一進さんの指示から最適なスキル/部門を判断して提案する。

| 指示の種類 | ルーティング先 |
|---|---|
| 「バナー作って」 | `/banner-park` |
| 「記事LP見て」 | `/kiji-cko` |
| 「動画作って」 | `/movie-kantoku` |
| 「Amazon対応して」 | `/amazon-captain` |
| 「リサーチして」 | `/research-park` |
| 「コンセプト考えて」 | `/concept-park` |
| 「提案書作って」 | `/proposal-park` |
| 「LP速度見て」 | `/lp-speed` |
| 「改善壁打ちしたい」 | `/park-kaizen` |

### 2. 今日のTo-Do提案

以下の情報源から今日やるべきことを整理:

1. `.claude/clients/` の各クライアント状況を確認
2. `git log --since="yesterday"` で昨日の作業を確認
3. `.claude/handoff/` に引き継ぎがあれば読む
4. 曜日に応じた定型タスクを提案

### 3. 進捗管理

- 各クライアントの現在フェーズを一覧表示
- 期限が近いタスクをハイライト
- 「次にやるべきこと」を優先度付きで提案

---

## 曜日別定型タスク

| 曜日 | 定型タスク |
|---|---|
| 月曜 | 週次振り返り準備 / 今週の優先順位整理 |
| 火〜木 | クリエイティブ制作 / クライアント対応 |
| 金曜 | `/weekly-review` / 来週の仕込み |

## 制約

- **やること**: タスク整理・ルーティング・進捗確認
- **やらないこと**: 各スキルの実行（それは各クルーの仕事）
- 一進さんの時間を奪わない。「どうしますか？」ではなく「こう整理した。どう？」で返す
- クライアント情報は clients/ から読む。推測しない
