## ワークフロー

### モード1: 台本単体生成

```
ユーザー: 「台本作って。テーマは〇〇、型は発声練習型」
  ↓
[Agent: sns-script-crew] テーマ・型を渡して台本生成
  ↓
品質スコア確認（75点以上？）
  ↓
結果をユーザーに見せる
```

### モード2: 台本5本セット（撮影用）

```
ユーザー: 「撮影用に台本5本作って」
  ↓
5本の型・テーマを設計（バリエーション最大化）
  ↓
[Agent: sns-script-crew] × 5回並列実行
  ↓
全台本の品質スコア確認
  ↓
HTMLレポート生成 → reports/sakura-scripts-vN.html
```

### モード3: 編集指示書生成

```
ユーザー: 「この台本の編集指示書作って」
  ↓
[Agent: sns-edit-crew] 台本データ + 素材リストを渡す
  ↓
編集指示書を返す
```

### モード4: 投稿準備

```
ユーザー: 「投稿準備して」
  ↓
[Agent: sns-post-crew] 台本データ + プラットフォーム指定を渡す
  ↓
説明文・ハッシュタグ・投稿時間・ウォームアップ手順を返す
```

### モード5: 分析

```
ユーザー: 「分析して」+ 投稿データ
  ↓
[Agent: sns-analytics-crew] データを渡す
  ↓
分析結果 + PDCA提案を返す
```

### モード6: 一気通貫

```
ユーザー: 「全部やって」
  ↓
Step 1: [Agent: sns-script-crew] 台本生成
Step 2: [Agent: sns-edit-crew] + [Agent: sns-post-crew] 並列実行
Step 3: 結果を統合して提示
```

### モード7: 撮影後パイプライン（🆕 ver.2.0）

撮影後の素材を受け取ったら、**編集スタイル**を選んでパイプラインを走らせる。

#### 🛤 3つのルート

**ルート A: 編集なし（素材 → ベストテイク → 投稿パッケージ）**
- 素材がそのまま使える想定（ワンカット撮影）
- さくら案件でも使える：1テイクの長回し素材をそのまま投稿
- 最速ルート。撮影データから30分で投稿準備完了

**ルート B: 手編集サポート（素材 → ベストテイク → カットリスト → 投稿パッケージ）**
- 澤田が自分でCapCut/Premiereでカットする前提
- AIは「どこを切るべきか」のカットリストJSONを生成
- カット後の動画を受け取って投稿パッケージ化
- **最初はこれがデフォルト**（素材の感触をつかむため）

**ルート C: AI自動編集（素材 → ベストテイク → edit_ai_sakura.py → 投稿パッケージ）**
- 全自動。人間介入なし
- 慣れてきたら使う。ルートBで蓄積したカットパターンを元に自動化

#### ルートA: 編集なし

```
Step 1: index_materials.py → materials.json
Step 2: sns-retake-crew → best_takes.json
Step 3: sns-post-crew → 投稿パッケージ（ベストテイクをそのまま使用）
Step 4: dashboard HTML
```

#### ルートB: 手編集サポート（DEFAULT）

```
Step 1: index_materials.py → materials.json
Step 2: sns-retake-crew → best_takes.json
Step 3: sns-edit-crew → カットリストJSON生成
        reports/edit-sheets/sakura_s{N}_cut_list.json
        + HTMLビジュアル版（編集指示書）
  ↓
  （澤田が手でカット編集 → edited_s{N}.mp4 を video-ai/sakura/edited/ に置く）
  ↓
Step 4: sns-post-crew → 投稿パッケージ（edited mp4を使用）
Step 5: dashboard HTML
```

#### ルートC: AI自動編集

```
Step 1-2: 同上
Step 3: edit_ai_sakura.py × 5本
Step 4: sns-post-crew → 投稿パッケージ
Step 5: dashboard HTML
```

#### モード7で使う外部ツール

| ツール | 用途 | どのルートで使う |
|---|---|---|
| `index_materials.py` | 素材インデックス生成 | A / B / C |
| `sns-retake-crew` | ベストテイク選定 | A / B / C |
| `sns-edit-crew` | カットリスト生成 | B |
| `edit_ai_sakura.py` | AI自動編集 | C |
| `sns-post-crew` | 投稿パッケージ生成 | A / B / C |
| `ffmpeg` / `ffprobe` | メタデータ・音量測定 | 全部 |

#### 最初のおすすめ

**初回撮影（4/11）はルートAかBで回す。** 素材の感触が見えたら、そこから蓄積したパターンでルートCの精度を上げる。

## HTML台本レポート出力

モード2（5本セット）実行時、`reports/sakura-scripts-vN.html` に美しいHTMLレポートを出力する。

**レポート要件:**
- 既存の `docs/sakura_proposal.html` と同じダークテーマ
- 各台本を1ページとして表示
- セリフは秒数タイムスタンプ付き
- 壺ポイントをゴールドでハイライト
- 品質スコアをレーダーチャート的に表示
- スマホで見やすい（撮影現場でスマホ確認するため）
- 印刷対応（page-break-after）

## 判断基準

- 「それ、バズるん？」が最初の問い。バズらないなら作り直す
- フックが弱い台本は絶対に通さない
- データがあるなら感覚より数字を信じる
- 迷ったら「さくらのキャラに合ってるか」で判断する
