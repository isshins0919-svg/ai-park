# Morning Routine Park v1.0
# テーマ: 時間効率 × 集中継続時間スケジューリング × 仕事ゴール管理

## 起動トリガー
Claude Code 起動時に自動実行（CLAUDE.md 初回起動アクション経由）

---

## 実行フロー

### STEP 0: 日付・曜日取得
- 今日の日付と曜日を確認
- OJTテーマをday_mapから選択（後述）

### STEP 1: カレンダー確認
Google Calendar MCPで以下を取得:
- `gcal_list_events` で今日の予定を全件取得
- 予定の開始・終了時刻から「余白ブロック」を自動計算
- 余白ブロック = 30分以上の連続した空き時間

出力フォーマット:
```
📅 今日の予定
  [時刻] タスク名
  ...

⏳ 余白ブロック
  [時刻帯] Xh Ym — [集中レベル: 高/中/低]
```

集中レベルの判定（時間帯 × 一般的エネルギーカーブ）:
- 09:00-11:00 → 高（Deep Work推奨）
- 11:00-12:00 → 中
- 13:00-14:00 → 低（Shallow Work推奨）
- 14:00-16:00 → 中〜高
- 16:00-18:00 → 中
- 18:00以降   → 低

### STEP 2: 仕事ゴール確認・設定
`.claude/work-goals.json` を読み込む。

**ファイルが存在しない or 空の場合:**
```
🎯 仕事ゴールがまだ設定されていません。
今やっている「仕事・プロジェクト」を教えてください。

例:
- LP制作（myskin face）
- Research Parkのスキル改善
- クライアントA の広告運用

いくつでも書いてOKです。それぞれにゴール（何をもって完了か）を一緒に決めます。
```

**ゴールが未設定の仕事がある場合:**
```
🎯 ゴール未設定の仕事があります:
- [仕事名]

それぞれ「何をもって完了か」を一言で教えてください。
```

**全ゴール設定済みの場合:**
```
🎯 今日の仕事ゴール
  ✅ [仕事名] — [ゴール]（進捗: [ユーザーが昨日設定した状態]）
  🔄 [仕事名] — [ゴール]（進捗: 進行中）
```

ゴール設定後、`work-goals.json` を更新して保存。

### STEP 3: 今日のスケジューリング提案
余白ブロック × 仕事ゴール × 集中レベルを組み合わせて最適配置を提案。

**スケジューリング原則（OJTの実践として毎日適用）:**
1. 集中レベル「高」の余白 → 最重要タスク or Deep Work
2. 集中レベル「中」の余白 → 通常タスク
3. 集中レベル「低」の余白 → 連絡・確認・軽作業
4. 連続集中時間の上限を90分として自動でバッファを挿入
5. 午前に1本、午後に1本「今日の最重要タスク」を配置

出力フォーマット:
```
📋 スケジューリング提案
  [時刻帯] → [仕事名: タスク内容]（集中: 高/中/低、所要目安: Xm）
  [時刻帯] → 🔋 休憩バッファ（10分）
  ...

💡 今日の最重要タスク: [タスク名]
```

### STEP 4: 今日のOJT
曜日に応じたOJTテーマを実行。

#### OJTテーマ ローテーション（月〜日）

**月曜: Deep Work設計**
```
🧠 OJT: Deep Work設計
今日の「重タスク」を特定して、集中ブロックに入れましょう。

チャレンジ:
今日やるべきタスクの中で、最も認知負荷が高いものを1つ選んでください。
それを何時〜何時のブロックに入れますか？（集中の高い時間帯に入れること）

→ [ユーザーの回答を待つ]
→ 回答後: Deep Workの理想時間（90分以内）と中断リスクを確認してフィードバック
```

**火曜: タスク認知負荷分類**
```
🧠 OJT: 認知負荷分類
今日のタスクを3分類してスケジュールの精度を上げましょう。

チャレンジ:
今日やること（思いつくもの全部）を書いてください。
→ 一緒に「重（創造・判断）/ 中（作業・確認）/ 軽（連絡・単純）」に分類します。

→ [ユーザーの回答を待つ]
→ 分類後: カレンダーの余白に最適配置を提案
```

**水曜: 時間見積もり精度チェック**
```
🧠 OJT: 見積もり精度チェック
昨日の予定vs実績を振り返って、見積もりの精度を上げましょう。

チャレンジ:
昨日、「思ったより時間がかかった」タスクはありましたか？
あれば: 何をやってたか / 予定 / 実際 を教えてください。

→ [ユーザーの回答を待つ]
→ 分析後: ズレのパターンを言語化、今日の見積もりに反映
```

**木曜: エネルギーカーブ配置**
```
🧠 OJT: エネルギーカーブ最適化
自分の集中力の波に合わせてタスクを配置しましょう。

チャレンジ:
あなたが最も集中できる時間帯はいつですか？
逆に眠くなる・ボーッとする時間帯は？

→ [ユーザーの回答を待つ]
→ 個人のエネルギーカーブを元にスケジュールを再提案
```

**金曜: 時間泥棒ハント**
```
🧠 OJT: 時間泥棒ハント
今週、時間を奪っていたものを1つ特定して来週に活かしましょう。

チャレンジ:
今週「これ、時間かかったな〜」or「やらなくてよかったかも」と感じたことはある？

→ [ユーザーの回答を待つ]
→ 時間泥棒のパターンを分析して、来週の対策を1つ提案
```

**土曜: バッファ設計**
```
🧠 OJT: バッファ設計
来週の予定にバッファを組み込んで、詰め込みすぎを防ぎましょう。

チャレンジ:
来週、すでに決まっている予定を教えてください。
→ 各タスクに「×1.5のバッファルール」を適用して余白を確保

→ [ユーザーの回答を待つ]
→ バッファ込みのスケジュール案を提案
```

**日曜: 週間振り返り & 来週設計**
```
🧠 OJT: 週間振り返り
今週の時間効率を自己採点して、来週の設計に活かしましょう。

振り返り3問:
1. 今週、最も集中できた仕事は？（なぜ集中できた？）
2. 今週、一番「もったいなかった」時間は？
3. 来週、1つだけ変えるとしたら何？

→ [ユーザーの回答を待つ]
→ 回答を元に来週の時間設計アドバイスを提案
```

```json
{
  "last_updated": "YYYY-MM-DD",
  "jobs": [

      "id": 1,
      "name": "仕事名",
      "goal": "何をもって完了か（一文）",
      "status": "active | completed | paused",
      "priority": "high | medium | low",
      "created_at": "YYYY-MM-DD",
      "notes": "メモ（任意）"
    }
  ]
}
```
