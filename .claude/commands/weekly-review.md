# Weekly Review Park ver.1.0 — 週次振り返り × 来週計画スキル

## 目的
月曜朝 or 金曜夕に走らせるだけで、今週の成果を自動集計し、
来週の優先順位と時間割を即出力する。「今週何やったっけ」をゼロにする。

---

## 発動トリガー

- 「週次レビューして」
- 「今週の振り返りして」
- 「weekly review」
- 「今週まとめて」
- 「来週の計画立てて」

---

## 実行手順

### Step 1: データ自動収集（並列実行）

以下を同時に取得する:

```bash
# 今週のgitコミット（月曜〜今日）
git log --oneline --since="last monday" --until="now"

# work-goals.jsonの現在状態
cat .claude/work-goals.json

# クライアントファイル全件（進行中案件の把握）
ls .claude/clients/
```

### Step 2: 今週の成果を整理

### Step 3: クライアントファイル自動更新（確認後）

振り返り完了後、以下を確認して更新する:

```
以下のクライアントファイルを更新してもいい？
- gungun.md: [更新内容]
- camicks.md: [更新内容]
→ 「はい」で一括更新
```

### Step 4: work-goals.json更新

完了したゴールのステータスを `completed` に変更し、
来週の新しいゴールを追加する。

---

## 週次レビューの問い（深掘り用）

振り返りを深めたい時は、以下の問いに答えてもらう:

```
1. 今週「思ったより時間がかかった」作業は何？なぜ？
2. 今週「これは効いた」と感じた瞬間は？
3. 来週もし1日しか使えないとしたら、何をやる？
4. クライアントに「もっと早く届けられた」と思う作業は？
```

---

## 月次レビュー（月末バージョン）

「月次レビューして」で起動。過去4週分のgitログを集計し、
月間の成果・売上貢献・スキル成長を整理する。

---

## 実行タイミングの推奨

| タイミング | 内容 |
|----------|------|
| 月曜朝（推奨） | 先週の振り返り + 今週の計画を一発で固める |
| 金曜夕 | 今週の締め + 来週の仕込み |
| 月末 | 月次レビュー（月間サマリー） |
