# Proposal Park ver.1.0 — 提案書・戦略資料 高速生成スキル

## 目的
「誰に・何を・なぜ提案するか」を投げるだけで、
Problem → Insight → Solution → ROI → Action Plan の構造で
クライアントが動きたくなる提案書を即生成する。

---

## 発動トリガー

- 「〇〇向けに提案書作って」
- 「〇〇の提案まとめて」
- 「proposal park」
- 「〇〇に提案したいんやけど整理して」
- 「〇〇の施策提案書作って」

---

## 提案書の種類（自動判定）

発動時に内容から種類を自動判定し、最適なテンプレを選択する。

| 種類 | 使うシーン | 例 |
|------|-----------|-----|
| **コスト削減型** | 原価・経費を下げる提案 | ameru原価削減 |
| **新規施策型** | 新しい取り組みの導入提案 | CRM導入 / AI活用 |
| **事業改善型** | 既存事業の改善・強化 | LP改善 / 広告最適化 |
| **事業継承型** | 経営移行・組織改革 | 澤田事業継承 |
| **投資判断型** | ROIベースの意思決定支援 | コンサル妥当性評価 |

## 実行手順

### Step 1: 提案の骨格をヒアリング（最大5問）

```
提案書を作るね！以下を教えて:

1. 提案相手: 誰に？（クライアント名 / 決裁者）
2. 提案内容: 何を提案する？（一言で）
3. 相手の課題: 相手が今困ってること・解決したいこと
4. こちらの仮説: 原因と解決策の仮説（あれば）
5. 使う場面: テレカン用スライド / 送付用PDF / 口頭説明メモ
```

クライアントファイルがあれば背景情報は自動補完し、
聞く質問を最小化する。

### Step 2: 提案書を生成

以下のフォーマットで出力する。

## 提案タイプ別の強調ポイント

### コスト削減型（ameru原価削減など）
- **効く構成**: 現状コスト → 削減ポイント特定 → 削減施策 → 削減額試算
- **必ず入れる**: 削減額の具体的な数字（年間〇〇万円）
- **過去事例**: ameru — 糸色数削減・パーツ設計変更・クリップ3→1個

### 新規施策型（CRM / AI導入など）
- **効く構成**: 現状の機会損失 → 施策の仕組み → 成功事例 → 実装ロードマップ
- **必ず入れる**: 競合や他業種の成功事例
- **過去事例**: ヨミテAI方針、ぐんぐんCRM戦略

### 投資判断型（コンサル妥当性など）
- **効く構成**: 投資額 vs 期待リターン → リスクシナリオ → 判断基準
- **必ず入れる**: やらない場合のコスト（現状維持のリスク）
- **過去事例**: 澤田AI導入コンサル妥当性評価

### 事業継承型（澤田コンサルなど）
- **効く構成**: 現状把握 → 移行ビジョン → ロードマップ → 短期アクション
- **トーン**: クライアント提案ではなく「一緒に考える」スタンス
- **注意**: 数字より「ストーリー」が刺さる

## 出力形式の選択

提案書を生成後、以下を聞く:

```
どの形式で出力する？
A. マークダウン（このまま）
B. HTMLスライド形式（NotebookLM風のビジュアル）
C. 口頭説明用メモ（箇条書き簡略版）
D. 全部
```

B（HTML形式）を選んだ場合: `reports/` フォルダに保存し、視覚的な提案スライドを生成する。
