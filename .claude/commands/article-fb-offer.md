## オファー評価フレームワーク

### 価格魅力度
| パターン | 評価 |
|---|---|
| 98%OFF（通常1万円→初回210円） | 最高（購買衝動 MAX） |
| 70%以上OFF | 高 |
| 50-70%OFF | 中 |
| 30%以下OFF / 割引なし | 低 |
| 通常価格のみ表示 | 最低 |

### 緊急性・希少性
| パターン | 評価 |
|---|---|
| 「本日23:59まで」+ 残り◯個 | 最強 |
| 期間限定（日付明記） | 強 |
| 「限定◯名」「今だけ」 | 中 |
| 緊急性なし | 弱 |

### 縛りなし設計
| パターン | 評価 |
|---|---|
| 定期縛りなし（いつでも解約OK）明記 | +15点 |
| 定期縛りあり（◯ヶ月継続）明記 | -10点 |
| 言及なし | 0点 |

## スコアリングロジック

```
offer_score =
  price_score      × 0.40   # 価格魅力度（割引率・演出）
  + urgency_score  × 0.25   # 緊急性・希少性
  + benefit_score  × 0.20   # 特典・ボーナス設計
  + binding_score  × 0.15   # 縛りなし設計

縛りなし明記ボーナス: +15点
```

## ワークフロー

```
Step 1: 記事テキスト受け取り
  ↓
Step 2: 価格情報を抽出（初回価格・通常価格・割引率）
  ↓
Step 3: 緊急性と希少性の実装状況を確認
  ↓
Step 4: 特典・ボーナスの内容と訴求強度を評価
  ↓
Step 5: 縛りなし設計の有無を確認
  ↓
Step 6: offer_score算出
  ↓
Step 7: 最優先改善1点を出力
  ↓
Step 8: JSONレポート出力
```

## 制約

- **やること**: 価格・割引・緊急性・特典・縛りなし設計の評価
- **やらないこと**: CTA文言評価（AGENT 04の担当）・信頼評価（AGENT 03の担当）
- 価格情報は必ず数値で抽出（「安い」等の定性評価のみはNG）
- 「縛りあり」を発見した場合は必ず明記する（重要な着地CVRリスク）
