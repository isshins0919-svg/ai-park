# Anonymize — 個人情報匿名化スキル

## 目的
顧客インタビューデータ・アンケートデータなどに含まれる個人情報を、
外部公開・共有可能な形式に匿名化する。

---

## 匿名化ルール（一進AIが必ず守る）

### 置き換える情報
| 情報の種類 | 置き換え方 |
|---|---|
| 氏名（顧客） | 顧客A、顧客B、顧客C … （出現順にアルファベット付番） |
| 氏名（解約者） | 解約者A、解約者B … |
| 氏名（社内担当者） | 担当者 or 社内担当者 |
| 居住地（市区町村以下） | 「関東在住」「近畿在住」など都道府県レベルに粗粒化 |
| メールアドレス | [非公開] |
| 電話番号 | [非公開] |
| 勤務先・職場名 | 「建設業」「事務職」など業種のみ残す |

### 残してよい情報（インサイトに必要）
- 年代（「40代」「50代」など丸める）
- 世帯年収レンジ（「800〜1,000万円台」など）
- 子供の性別・学年（氏名は削除）
- 購入動機・発言内容（名前を紐づけない形で）
- SNS利用傾向・購買行動

---

## 発動タイミング

以下のいずれかの場合は**自動的に匿名化**して出力する：
1. HTMLレポート・スライドを作成するとき
2. ドキュメント・マークダウンにインタビュー内容を記載するとき
3. GitHubにpushするファイルに個人情報が含まれるとき
4. ユーザーから「匿名化して」と指示されたとき

---

## 匿名化マッピングの管理

同一ドキュメント内では匿名化IDを一貫させる。
（同じ人物が顧客Aならば、そのドキュメント全体で顧客A）

複数ドキュメントにまたがる場合は、
都度セッション内でマッピングを宣言してから作業する。

---

## 実行コマンド例

ファイルの匿名化をユーザーから依頼された場合：
1. ファイルを読み込む
2. 個人情報箇所を特定してリストアップ
3. マッピングを宣言（「黒岩様 → 顧客A様」など）
4. sed / 一括置換で処理
5. 置換後に残存個人情報がないかgrepで確認
6. 完了報告
