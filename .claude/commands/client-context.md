# Client Context Manager — クライアントコンテキスト読み込みスキル

## 目的
クライアント名を受け取り、該当クライアントの背景・進行中案件・過去施策・KPI・注意事項を一瞬で読み込み、作業に即座に入れる状態にする。

---

## 発動トリガー（これらのフレーズで自動発動）

- 「〇〇案件やって」
- 「〇〇の続きから」
- 「〇〇向けに〜して」
- 「〇〇のコンテキスト読んで」
- 「client load 〇〇」

---

## 実行手順

### Step 1: クライアントファイル検索

`.claude/clients/` 配下から、指定されたクライアント名に一致するファイルを読み込む。

**名前の揺らぎ対応（例）**:

// ヨミテグループ
- 「ヨミテ」「えみさん」 → `yomite.md`（親会社。ブランドをまたぐ話・会社方針はここ）
- 「gungun」「ぐんぐん」「伸長」 → `gungun.md`（ヨミテ傘下）
- 「onmyskin」「オンマイスキン」「onmy」 → `onmyskin.md`（ヨミテ傘下）
- 「proust」「プルースト」 → `proust.md`（ヨミテ傘下）

// 澤田グループ
- 「澤田」「澤田さん」「父の会社」「事業継承」「次期社長」「sawada」「澤田本体」「コンサル」 → `sawada-co.md`（親会社・事業継承コンサル統合。一進さん自身が入る会社）
- 「camicks」「キャミックス」「靴下」「靴下Amazon」 → `camicks.md`（Camicks靴下事業・Amazon出店支援）

// その他
- 「ameru」「アメル」 → `ameru.md`
- 「mauri」「マウリ」 → `mauri.md`
- 「rkl」 → `rkl.md`

**グループ案件のルール**:
- ブランド名を指定 → ブランドファイルを読む
- 「えみさん全体」「澤田さん全体」「会社方針」など複数ブランドをまたぐ話 → 親会社ファイルも合わせて読む

ファイルが存在しない場合 → Step 3（新規作成）へ

### Step 1.5: データソース把握（重要）

clients/*.md を読み込んだら、以下を必ず実行:

1. **MD内の「データソース」セクションを確認** — そのクライアントのDesktop配下データが列挙されている
2. **`~/Desktop/_clients/<client>/` が存在するか Glob で確認** — 無ければスキップ
3. **ユーザーの要求に応じて必要なファイルだけRead** — 全件Readは禁止（コンテキスト無駄）
   - 「経営計画見て」→ 経営計画Excelを選んでRead
   - 「AI研修の続き」→ 研修提案書PDFを選んでRead
   - 要求が曖昧な時は「どのデータ見る？」と聞く
4. **データソース欄に無いファイルが必要そうな時** → Globで探して、見つけたら「未登録ファイル `xxx` も見る？登録する？」と確認

未登録ファイル検出用スクリプト: `./scripts/check_client_data.sh <client>`

### Step 2: コンテキスト読み込み完了報告

以下のフォーマットで読み込み完了を報告し、即座に作業モードに入る:

```
📁 [クライアント名] 読み込み完了

■ 事業: [一行サマリ]
■ 進行中: [進行中プロジェクト名 + ステータス]
■ 注意: [特記事項があれば1〜2行]

→ 何やる？
```

### Step 3: 新規クライアントの場合

ファイルが存在しなければ、以下の情報を聞いて `.claude/clients/{name}.md` を新規作成する:

```
[クライアント名] のファイルがまだないで！
簡単に教えて:
1. ブランド名・業種
2. メイン商品
3. 今やってること
4. 気をつけること（あれば）
```

収集後、テンプレートに従ってファイルを作成し確認を取る。

## クライアントファイル更新ルール

### 自動更新タイミング
作業完了時に、以下を対象ファイルに追記・更新する:
- 完了した施策（日付付きで）
- 判明したKPI・数字
- 新たに分かった特記事項
- 次のアクション

### 更新コマンド
「〇〇更新して」または作業完了後に自動で確認:
```
→ [クライアント名].md を更新してもいい？
  追記内容: [更新内容の要約]
```
