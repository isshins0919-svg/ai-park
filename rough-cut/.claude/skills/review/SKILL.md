# Skill: review

テロップテキストの品質チェック + 誤字脱字修正。

## 入力

`stt_result.json` の sentences を読み込む。

## 実施内容

1. **テキスト通し読み**: 全 sentences のテキストを順に読み、動画の流れを確認
2. **誤字脱字チェック**: STT の誤認識、不自然な表現を特定
3. **表記正規化チェック**: 下記の共通ルールに照らして corrections に追加
4. **品質チェック**: カット後のテキストが自然に繋がるか確認

## 共通正規化チェックリスト

以下は step08 の自動正規化で処理されるが、漏れがないか確認する:

### 数字 (漢数字→算用数字)
- 「三十八度」→「38度」
- 「千九百円」→「1,900円」
- 「百個」→「100個」
- 「十六万」→「16万」
- 「一万円」→「1万円」
- **注意**: 助数詞のない漢数字は自動変換されない (「一般」「三角」等)。文脈で判断

### 固有名詞
- 「YOUTUBE」→「YouTube」
- 「INSTAGRAM」→「Instagram」
- 「TIKTOK」→「TikTok」
- 話者名・チャンネル名: project 情報から正しい表記を確認

### STT 誤認識パターン
- カタカナ語の表記揺れ (マネタイズ/モニターズ/モネタイズ 等)
- 英語の聞き間違い
- 同音異義語

## 判断基準

修正すべきもの:
- 明らかな STT 誤認識 (例: 固有名詞の聞き間違い)
- 自動正規化で漏れた数字表記
- 意味が通らない箇所

修正しないもの:
- 話者の言い回しの癖 (口語表現はそのまま)
- 文法的に厳密でなくても意味が通る表現

## 出力形式

確認結果を `review.json` として保存する:

```json
{
  "corrections": {
    "間違ったテキスト": "正しいテキスト"
  },
  "quality_notes": [
    "全体的に流暢。特に問題なし"
  ]
}
```

corrections は step08 で `--review` 引数経由でテロップに自動反映される。
自動正規化で処理済みの項目は corrections に入れなくてよい。

## 手順

1. `stt_result.json` を Read で読む
2. sentences のテキストを順に通し読み
3. 誤字脱字・表記正規化の漏れ・品質問題を確認
4. 結果を `review.json` として Write で保存
